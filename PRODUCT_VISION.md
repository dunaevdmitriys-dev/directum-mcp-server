# Directum MCP Server — Продуктовое видение

**Версия:** 1.0 | **Дата:** 13 марта 2026

---

## 1. Ключевой тезис

MCP-сервер — это не «чат-бот для Directum», а **интеллектуальный слой, превращающий Directum RX из системы учёта в систему действия.**

## 3. Эволюция продукта: от CRUD к интеллекту

### Уровень 1 — «Удобно» (Q2 2026)
Делает существующие операции быстрее.
- `find_docs` с NL-поиском
- `summarize` — суть документа за 30 секунд
- `bulk_complete` — массовое выполнение
- `fix_package` — автоисправление ошибок

### Уровень 2 — «Умно» (Q3 2026)
Аналитика, которую раньше мог делать только человек.
- `bottleneck_detect` — где застревают маршруты
- `deadline_risk` — предсказание просрочки
- `team_workload` — нагрузка подразделения
- `compliance_check` — соответствие регламентам

### Уровень 3 — «Незаменимо» (Q4 2026)
Невозможно без LLM.
- `meeting_to_tasks` — протокол → поручения
- `email_to_document` — email → карточка документа
- `contract_review` — анализ рисков в договоре
- `report_builder` — отчёт на естественном языке

## 4. Именование инструментов

| Текущее | Предлагаемое | Почему |
|---------|-------------|--------|
| `validate_package` | `check_package` | интуитивнее |
| `read_mtd` | `inspect` | универсальнее |
| `search_documents` | `find_docs` | короче, глагол первым |
| `my_assignments` | `my_tasks` | привычнее |
| `complete_assignment` | `complete` | контекст очевиден |
| `create_task` | `send_task` | в Directum задача «отправляется» |

Русские алиасы: `найди`, `мои_задания`, `согласуй`

## 8. Риски

| Риск | Уровень | Митигация |
|------|---------|-----------|
| LLM видит конфиденциальные данные | Высокий | On-premise, metadata_only режим, DLP |
| Галлюцинации → неверные решения | Высокий | Данные только из API, ссылки на источник, подтверждения |
| Злоупотребление правами | Средний | Аутентификация = пользователь, rate limiting, audit-лог |
| Зависимость от одного LLM-провайдера | Средний | MCP = протокол, поддержка GigaChat/YandexGPT/Llama |
| Сопротивление пользователей | Низкий | Начинаем с pain-driven (валидация пакетов — реальная боль) |

## 10. Принципы продукта

1. **Факты, а не выдумки.** Все данные — из Directum RX через API.
2. **Безопасность по умолчанию.** On-premise, права пользователя, подтверждения, audit-лог.
3. **Постепенное усложнение.** DevTools → Runtime → Analytics → Platform.
4. **Open core.** Community создаёт экосистему, Enterprise монетизирует.
5. **Доменная глубина.** Не «MCP для документов», а эксперт по ГОСТ, 63-ФЗ, УПД и маршрутам.
