# Разработка справочников

На уровне платформы к базовым типам справочников относятся:

- DatabookEntry – справочник, на основе которого создано большинство справочников в системе. Примеры наследников: Населенные пункты , Журналы регистрации ;
- User – справочник пользователей системы. Он и его наследники используются для назначения прав доступа. Пример наследника: Сотрудники ;
- Group – справочник с группами пользователей системы. Он и его наследники используются для назначения прав доступа. Примеры наследников: Подразделения , Группы регистрации .

## Особенности справочников

Подробнее о порядке создания типа справочника см. в разделе «Типы сущностей» .

При выборе наименования для справочника рекомендуется:

- указывать имя в единственном числе;
- не использовать транслитерацию. Например, назовите тип справочника «Department» вместо «Оtdel»;
- ориентироваться на названия типов сущностей в базовом решении . Например, если в тип сущности добавляется свойство-ссылка на справочник «Контрагенты» (Company), то назовите свойство «Company» вместо «Organization».

В редакторе типа справочника задаются:

- основные параметры ;
- свойства , которые выносятся на форму ;
- действия , которые добавляются на ленту ;
- списки типов сущностей;
- стили отображения записей.

Какие типы справочников можно перекрывать , а от каких – создавать наследников , можно посмотреть в среде разработки.

## События типов справочников

Для типа справочника кроме основных серверных и клиентских событий дополнительно доступны события:

Серверные события | Разделяемые события
UI-фильтрация (UIFiltering) | Получение информации о подписавшем (SignatoryInfoObtaining)
Показ подсказки (GetDigest) (только для наследников Sungero.CoreEntities.User )

Также в каждом типе сущности можно задать события свойств и обработчики действий .
