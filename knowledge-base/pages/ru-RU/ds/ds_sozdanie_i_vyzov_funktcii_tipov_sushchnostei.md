# Создание функций типов сущностей

Предположим, что для входящего письма необходимо искать дубли документов и показывать их в списке. Для выполнения задачи необходимо создать серверную функцию, которая ищет дубли документа, затем вызвать ее в обработчике действия.

Для этого:

- 1. В редакторе типа сущности IncomingLetter в группе «Функции» рядом со ссылкой Серверные нажмите на кнопку .
- 2. В открывшемся редакторе измените имя созданной функции, добавьте комментарий с описанием и напишите ее код. При необходимости измените тип возвращаемого значения, входные параметры, атрибуты и модификаторы функции.

```csharp
//// <summary>
/// Серверная функция для получения дублей входящего письма.
/// </summary>
/// <param name="letter">Письмо для проверки.</param>
/// <param name="documentKind">Вид письма.</param>
/// <returns>Письма, дублирующие текущее.</returns>
[Remote(IsPure = true)]
public static IQueryable<IIncomingLetter> GetDuplicate(IIncomingLetter letter, Sungero.Docflow.IDocumentKind documentKind)
{
return IncomingLetters.GetAll()
.Where(l => documentKind == l.DocumentKind);
.Where(l => !Equals(letter, l));
}
```

- 3. Сохраните изменения.
- 4. Для действия добавьте обработчик события Выполнение . Для этого в Development Studio в группе «Обработчики» рядом со ссылкой Выполнение нажмите на кнопку .
- 6. В открывшемся редакторе добавьте вызов серверной функции.
- Чтобы вызвать серверную функцию используйте формат:
- Functions.<Тип сущности>.Remote.<Имя функции>(<Параметры>) . В формате используется Remote , потому что из клиентского кода вызывается серверная функция. Формат вызова других функций описан ниже .

```csharp
/// <summary>
/// Клиентская функция для формирования списка дублей входящего письма.
/// </summary>
public virtual void ShowDuplicates(Sungero.Domain.Client.ExecuteActionArgs e)
{
var duplicates = Functions.IncomingLetter.Remote.GetDuplicate(_obj, _obj.DocumentKind, _obj.BusinessUnit, _obj.InNumber, _obj.Dated, _obj.Correspondent);
if (duplicates.Any())
duplicates.Show();
else
Dialogs.NotifyMessage(IncomingLetters.Resources.DuplicateNotFound);
}
```

- 7. Сохраните изменения.

## Формат вызова функций типов сущностей

Функции вызываются по формату Functions.<Имя типа сущности>.<Имя функции>(<параметры>) . При этом вызов функции с атрибутом:

- Remote , то есть когда серверная функция вызывается в клиентском коде, происходит в формате:
- Functions.<Имя типа сущности>. Remote .<Имя функции>(<Параметры>);
- Public , то есть когда функция вызывается в функции другого модуля, происходит в формате:
- PublicFunctions .<Имя типа сущности>.<Имя функции>(<Параметры>).

Формат вызова можно сократить, если функция вызывается:

- из другого типа сущности. Functions можно не указывать;
- из того же типа сущности, в котором она создана. Functions.<Имя типа сущности> можно не указывать.
