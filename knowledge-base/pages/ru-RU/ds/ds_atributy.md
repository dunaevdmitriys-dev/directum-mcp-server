# Атрибуты

Атрибуты предназначены для указания способа вызова функций. Например, чтобы в клиентском коде вызывать серверную функцию или из функции одного модуля вызывать функцию другого. Атрибуты указываются в квадратных скобках перед объявлением функции.

ПРИМЕЧАНИЕ. Чтобы использовать несколько атрибутов, укажите их через запятую перед объявлением функции. Например, укажите Remote и Public , если планируется вызывать функцию в клиентском коде и использовать в других модулях.

Существуют следующие атрибуты:

Атрибут | Когда указывается
Public | При использовании функции, структуры или константы в других модулях или решениях
Public(WebApiRequestType = RequestType.Get)] и [Public(WebApiRequestType = RequestType.Post) | При создании функции интеграции
Remote | При создании серверной функции , которую нужно вызвать в клиентском коде
Hyperlink | При создании клиентской функции , которая выполняется при нажатии на ссылку
LocalizeFunction | Может использоваться при добавлении локализации для имени и описания при создании клиентской функции , связанной с действием обложки модуля, а также для функции контрола состояния , используемой в редакторе формы в проводнике
Converter | При создании серверной функции сущности, которая преобразуют один объект в другой по заданной логике
ExpressionElement | При создании функции вычисляемых выражений

## Public

Указывается, если функцию, структуру или константу нужно использовать в других модулях или решениях. Атрибут можно использовать в функциях инициализации.

## [Public(WebApiRequestType = RequestType.Get)] и [Public(WebApiRequestType = RequestType.Post)]

- Указываются для создания функции интеграции .

ПРИМЕЧАНИЕ. Атрибуты можно использовать только в функциях модуля.

## Remote

Указывается при создании серверной функции, которую нужно вызвать в клиентском коде.

Для атрибута Remote можно дополнительно указывать параметры:

- PackResultEntityEagerly . Добавляется, чтобы функция возвращала сущность вместе со свойствами ( «жадная» загрузка ). Например, чтобы при создании задачи сразу за один запрос загружались значения полей и вложения. Возможные значения: true , false . Значение по умолчанию false . Указывается в формате: [Remote(PackResultEntityEagerly = true)] ;
- IsPure . Рекомендуется использовать для оптимальной работы функций, которые не вносят изменения в данные , например, получают их или проверяют условия. Возможные значения: true , false . Значение по умолчанию false . Указывается в формате: [Remote(IsPure = true)] ;
- UseTransaction . Отвечает за выполнение функции в одной транзакции. Возможные значения :

• true – процесс сохранения нескольких сущностей проходит в одной транзакции. Ошибка при сохранении одной сущности отменит транзакцию, и все последующие изменения не сохранятся. Значение по умолчанию;

• false – процесс сохранения нескольких сущностей проходит в несколько транзакций. Рекомендуется использовать значение, если нужно сохранить большое количество сущностей, например в цикле. Тогда ошибка при сохранении одной сущности не повлияет на сохранение остальных.

- Атрибут указывается в формате: [ Remote(UseTransaction = true)] .

Важно. Если вызвать серверную функцию с атрибутом Remote в событиях «Показ формы» , «Возможность выполнения» или «Обновление формы» , то результаты ее выполнения кэшируются. При ее повторном вызове с теми же входными параметрами функция возвращает значение из кэша. Оно хранится, пока открыта карточка сущности или не выполнено ее принудительное обновление. Таким образом снижается нагрузка на работу веб-сервера.

Пример 1. Создание задачи на ознакомление с документом

```csharp
/// <summary>
/// Создать задачу на ознакомление с документом.
/// </summary>
/// <param name="document">Документ, который отправляется на ознакомление.</param>
/// <returns>Задача на ознакомление по документу.</returns>
[Remote(PackResultEntityEagerly = true), Public]
public static IAcquaintanceTask CreateAcquaintanceTask(IOfficialDocument document)
{
var newAcqTask = AcquaintanceTasks.Create();
newAcqTask.DocumentGroup.OfficialDocuments.Add(document);
return newAcqTask;
}
```

Пример 2. Получение подразделения по идентификатору

```csharp
/// <summary>
/// Получить подразделение по id.
/// </summary>
/// <param name="id">Id.</param>
/// <returns>Подразделение.</returns>
[Public, Remote(IsPure = true)]
public static IDepartment GetDepartmentById(long id)
{
return Departments.Get(id);
}

// Пример вызова функции из клиентского кода модуля Shell:
var department = Sungero.Company.PublicFunctions.Module.Remote.GetDepartmentById(departmentId);
```

Пример 3. Получение списков ознакомления

```csharp
/// <summary>
/// Получить списки ознакомления.
/// </summary>
/// <returns>Списки ознакомления.</returns>
[Public, Remote(IsPure = true)]
public IQueryable<IAcquaintanceList> GetAcquaintanceLists()
{
return AcquaintanceLists.GetAll()
.Where(a => a.Status == Sungero.RecordManagement.AcquaintanceList.Status.Active);
}
```

- Hyperlink

Указывается при создании клиентской функции , которая выполняется при нажатии на ссылку. Используйте атрибут, например, когда по ссылке необходимо открыть объект внешней системы, или в текст задачи добавить ссылку, при нажатии на которую откроется список или карточка типа сущности.

ПРИМЕЧАНИЕ . Атрибут можно использовать только в функциях модуля.

Для корректного отображения ссылки локализуйте ее название. В параметрах атрибута укажите имя добавленного ресурса:

```csharp
[Hyperlink(DisplayNameResource="<Имя ресурса>")]
```

Для формирования ссылки на функцию используйте класс Hyperlinks :

```csharp
var url = Hyperlinks.Functions.<Имя клиентской функции>();
```

Формат формируемой ссылки:

<Адрес обработчика гиперссылок>?module.id=<Идентификатор модуля>&module.func=<Имя клиентской функции>&<Параметр функции 1>=<Значение 1>&<Параметр функции 2>=<Значение 2>...

ПРИМЕЧАНИЕ. Адрес обработчика гиперссылок задается в параметре HYPERLINK_SERVER конфигурационного файла config.yml.

Пример ссылки:

http://rx.company.ru/sungero?module.id=243c2d26-f5f7-495f-9faf-951d91215c33&module.func=Unsubscribe&UserId=13&subscription=ActionItemDeadline

## LocalizeFunction

Используйте атрибут, чтобы локализовать имя и описание функции, которые отображаются в редакторах форм, обложек и др:

Например, он используется при создании клиентской функции , связанной с действиями обложек модулей, а также для серверной функции без параметров, которая возвращает модель контрола состояния StateView .

Атрибут указывается в формате [LocalizeFunction("<Отображаемое имя функции>", "<Описание>")] .

Пример. Получение списка сотрудников

```csharp
/// <summary>
/// Получить список всех сотрудников.
/// </summary>
/// <returns>Список всех сотрудников.</returns>
[LocalizeFunction("Все сотрудники", "Возвращает список всех сотрудников.")]
public IQueryable<IEmployee> EmployeesAll()
{
return Employees.GetAll();
}
```

Вместо имени и описания функции также можно передавать ресурсы :

```csharp
[LocalizeFunction("EmployeesAll_ResourceKey", "EmployeesAll_DescriptionResourceKey")]
public IQueryable<IEmployee> EmployeesAll()
{
return Employees.GetAll();
}
```

## Converter

Указывается при создании серверных функций сущности, которые преобразуют один объект в другой по заданной логике. Используйте атрибут, например, если в шаблоне документа с автозаполняемыми полями нужно настроить параметры для изменения значений: преобразования числа в строку, склонения по падежам, преобразования строки (изменение регистра, взятие первой буквы и т.д.).

В атрибуте можно указывать только один параметр:

[ Converter ( "<Имя ресурса>" )]

Для создания функции-конвертера добавьте серверный метод с модификатором public и заключите его в атрибут Converter , а также ресурсы для отображаемого значения функции-конвертера в системе .

ПРИМЕЧАНИЕ. Атрибут можно использовать только в статических функциях типов сущностей.

Пример. Капитализация слова (написание с прописной буквы)

В редакторе кода в перекрытом документе OfficialDocument (Официальный документ) добавьте ресурс для отображения в системе значения «С прописной буквы» функции капитализации слова.

Затем добавьте ресурс для функции:

```csharp
/// <summary>
/// Получение слова, написанного с большой буквы.
/// </summary>
/// <param name="target">Слово.</param>
/// <returns>Слово, написанное с большой буквы.</returns>
[Converter("Capitalize")]
public static string Capitalize(string target)
{
if (string.IsNullOrEmpty(target))
return target;
return target.Substring(0, 1).ToUpper(CultureInfo.CurrentCulture) + (target.Length > 1 ? target.Substring(1) : string.Empty);
}
```

## ExpressionElement

Указывается при создании функций вычисляемых выражений в формате:

[ ExpressionElement ( "<имя функции>" , "<Описание функции 1>" , "<Описание параметра 2>" , "<Описание параметра 3>" , "<Описание параметра N>" )] .

Где:

- < Имя функции > – название функции, отображаемое в выпадающем списке при заполнении вычисляемого выражения. Обязательный параметр;
- < Описание функции > – подсказка по использованию функции;
- < Описание параметра 2…N > – подсказки с описанием параметров. Описания задаются в порядке следования параметров.

ПРИМЕЧАНИЕ . Первый параметр подставляется автоматически, задавать для него описание не нужно. Вместо этого укажите только двойные кавычки "". Описание задается со второго параметра.
