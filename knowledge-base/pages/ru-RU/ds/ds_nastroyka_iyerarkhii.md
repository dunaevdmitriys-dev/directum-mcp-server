# Настройка иерархии в папке

В папке можно отображать объекты в виде иерархического списка. Например, для справочника Наши организации , в котором есть головные и дочерние организации. Для этого задайте параметры:

Использовать иерархию . Если флажок установлен, содержимое папки отображается в виде древовидной структуры. Для настройки иерархии заполните поля:

- *Свойство-родитель , по которому строится иерархия. Возможные значения: свойства-ссылки типа сущности, указанного в поле Основное содержимое .
- Например, чтобы настроить иерархию для записей справочника Наши организации , в выпадающем списке укажите свойство HeadCompany (Головная орг.).
- *Отображать в . Столбец, в котором отображается иерархия. Возможные значения: свойства-ссылки или свойства-строки типа сущности, указанного в поле Основное содержимое .
- СОВЕТ. Иерархию удобно отслеживать по названию, в таком случае можно выбрать свойство Name (Наименование);
- *Получать записи . Способ выполнения события «Получение данных» . Возможные значения:

• Верхнего уровня . Событие применяется только при первоначальной загрузке списка для родительских записей. Например, условия события применяются только для головной организации в справочнике Наши организации .

• Всей иерархии . Событие применяется на всех уровнях иерархического списка: родителях и их потомках. В этом случае условия применяются и для головной, и для дочерних организаций справочника Наши организации .

- При первоначальной загрузке иерархического списка также применяется событие «Предварительная фильтрация» для обоих типов значений поля Получать записи .

Включать перемещение записей в иерархическом списке . Если флажок установлен, то можно перемещать записи иерархического списка между узлами с помощью механизма Drag&Drop. При этом ее свойство-родитель изменяется и сохраняется с отработкой событий, в том числе выполняется валидация . Если по ее результатам сохранение записи невозможно, перемещение не выполняется и в проводнике отображается соответствующее сообщение.

ПРИМЕЧАНИЕ. При необходимости вы можете добавить собственные ограничения работы Drag&Drop в иерархическом списке, прописав код в событии «До сохранения» .

## Особенности работы с иерархическими списками

При настройке иерархического списка в папке необходимо учитывать особенности:

- в параметрах папки в поле Область отображения необходимо указывать значение Модуль , т.к. в папках потока отображаются только те записи, в которых пользователь является исполнителем или инициатором;
- событие «Предварительная фильтрация» применяется для обоих типов значений поля Получать записи только при первоначальной загрузке иерархического списка;
- если в поле Получать записи выбрано значение Верхнего уровня , список экспортируется только для родительских записей. Для значения Всей иерархии выгружается весь список без учета иерархии;
- недоступна группировка записей в списке;
- недоступно компактное представление списка;
- в программном коде необходим контроль цикличности сущностей. Например, ввести запрет на сохранение записи при работе со списком наших организаций, если дочерняя организация становится родительской записью для головной.

• событие «UI-фильтрация» вызывается при открытии папки и развертывании узлов иерархического списка.

## Пример ограничения работы Drag&Drop

Если в иерархических списках нужно добавить собственные ограничения работы механизма Drag&Drop, дополните логику валидация . Для этого в событии сущности До сохранения воспользуйтесь параметром AfterDnD и пропишите необходимый код. Например, чтобы отключить Drag&Drop в иерархическом списке:

```csharp
public override void BeforeSave(Sungero.Domain.BeforeSaveEventArgs e)
{
if (e.Params.Contains("AfterDnD"))
{
e.AddError("Нельзя перемещать запись с помощью механизма Drag&Drop");
}
}
```

## Пример создания дочерних записей в иерархическом списке

Чтобы в иерархическом списке при создании запись автоматически становилась дочерней для текущей выделенной записи, необходимо в обработчике события «Создание» написать код, задающий значение свойства-родителя создаваемой сущности. Для этого доступны дополнительные параметры:

- ActiveEntityId – идентификатор выделенной записи;
- ActiveEntityType – тип выделенной записи.

Например, в справочнике Наши организации для иерархии у новых создаваемых записей можно реализовать автоматическое заполнение свойства HeadCompany (Головная орг.):

```csharp
public override void Created(Sungero.Domain.CreatedEventArgs e)
{
var parentEntityId = 0;
if (e.Params.TryGetValue("ActiveEntityId", out parentEntityId))
{
_obj.HeadCompany = BusinessUnits.Get(parentEntityId);
}
base.Created(e);
}
```
