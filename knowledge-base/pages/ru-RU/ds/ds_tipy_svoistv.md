# Типы свойств

Тип свойства определяет, какие данные могут быть значением свойства. Также тип задает способ отображения значения и дополнительные параметры свойства.

Свойства могут быть следующих типов:

Тип свойства | Описание
Коллекция | Используйте для работы с набором записей, у которых есть свои свойства. Например, для хранения информации о выдаче и возврате документов. Тип устанавливается автоматически при создании свойства-коллекции
Логическое | Используйте для свойств, которые могут иметь значения True и False , например для свойства поручения На контроле
Бинарные данные | Используйте для работы с данными небольшого размера. Например, для работы с текстовыми файлами. ПРИМЕЧАНИЕ. Значение свойства задается только программно
Дата | Используйте для работы с датой и временем
Вещественное | Используйте для работы с вещественными числами
Перечисление | Используйте, если возможных значений у свойства немного и они заранее известны
Картинка | Используйте для работы с графическими данными небольшого размера. Например, для хранения фотографии в карточке сотрудника. ПРИМЕЧАНИЕ. Поддерживаются следующие форматы изображений: BMP , GIF , JPEG , PNG , TIFF , WMP , ICO
Целое | Используйте для работы с целыми числами
Идентификатор | Число типа long. Предназначен только для хранения идентификаторов сущностей. В остальных случаях для хранения чисел достаточно меньшего диапазона значений, поэтому используйте тип «Вещественное» или «Целое»
Ссылка | Используйте для работы со ссылками на типы сущностей, возможными значениями свойства будут сущности указанного типа. Например, если свойство является ссылкой на справочник Сотрудники , то возможными значениями будут записи этого справочника
Строка | Используйте для работы со строками. Например, для указания краткого содержания документа
Текст | Используйте для работы с текстовыми данными. Например, для хранения примечаний к документам и записям справочников
Бинарные данные в хранилище | Используйте для работы с данными больших размеров. Например, для хранения содержимого электронных документов. ПРИМЕЧАНИЕ. Значение свойства задается только программно

## Коллекция

Отображение свойства в карточке сущности зависит от типа контрола , который настраивает аналитик через представления форм в веб-клиенте . Чтобы при добавлении или удалении записи из коллекции дополнительно выполнялась некоторая логика, в редакторе типа сущности на вкладке Свойства добавьте обработчики событий «Добавление в коллекцию» и «Удаление из коллекции» .

## Дата

В редакторе типа сущности на вкладке Свойства в поле *Формат даты дополнительно укажите формат отображения даты и времени. Возможные значения: Только дата , Дата и время , Год и месяц , Только год . Чтобы программно проверить, в каком формате отображаются значения свойств типа «Дата», используйте перечисление CommonLibrary.DateTimeFormat .

## Ссылка

Отображение свойства-ссылки зависит от способа выбора значения для типа сущности, на который настроена ссылка. Способ выбора значения указывается на вкладке Главная редактора типа сущности. Может отображаться как: • выпадающий список: • отдельная форма-список, которая открывается при нажатии на кнопку : • форма с древовидным списком, которая используется для выбора сотрудников из организационной структуры: Чтобы задать тип сущности, записи которого будут возможными значениями свойства, в редакторе типа сущности на вкладке «Свойства» заполните поля: • *Тип свойства . В выпадающем списке выберите значение Ссылка ; • *Тип сущности . Сущности данного типа будут значениями свойства. Например, в справочнике Город для свойства Регион можно настроить заполнение с помощью типа сущности Регион ; • Отображать как список при выборе . Флажок доступен только для типа сущности Sungero.CoreEntities.Recipients и его наследников. Если флажок установлен, то для выбора значения будет использоваться форма-список; • Отображать как выпадающий список при выборе . Флажок доступен только для типа сущности Sungero.CoreEntities.Recipients и его наследников. Если флажок установлен, то в веб-клиенте для выбора значения используется выпадающий список, даже если в редакторе типа сущности в поле Способ выбора значения указано значение Форма-список . При этом выбор значений из формы с древовидным списком недоступен. Если нужно использовать форму с древовидным списком, то снимите флажки Отображать как список при выборе и Отображать как выпадающий список при выборе , а также проверьте, что в редакторе типа сущности в поле Способ выбора значения указано значение Форма-список . Если необходимо настроить фильтрацию списка возможных значений свойства, задайте обработчик события Фильтрация выбора из списка .

## Строка

В редакторе типа сущности на вкладке Свойства в поле *Длина строки дополнительно укажите максимально возможное количество символов в строке в диапазоне от 1 до 1000.
