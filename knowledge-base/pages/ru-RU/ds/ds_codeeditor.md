# Программный код

В среде разработки для написания программного кода используется язык C# с применением объектной модели и учетом ограничений, которые накладываются ядром Directum RX.

Совет. При возникновении вопросов по синтаксису C#, обращайтесь, например, к справочнику на сайте Microsoft.

Программный код пишется в стороннем редакторе кода, например VS Code. К коду можно перейти из редакторов элементов разработки или открыть нужный файл с кодом сразу в стороннем редакторе.

В программном коде используются следующие элементы:

- обработчики – фрагменты кода, реализующие определенную логику в ответ на события типов сущностей, модулей, блоков, групп вложений, фоновых процессов и т.д. Например, чтобы при создании документа заполнялась его дата создания;
- функции – именованные последовательности операторов, предназначенные для многократного использования, например, для создания документов;
- структуры – объекты, которые состоят из нескольких логически связанных между собой полей разного типа. Например, в структуре можно хранить персональную информацию о сотруднике: ФИО, дату рождения и паспортные данные;
- константы – заданные значения, которые не изменяются при выполнении кода. Используются, например, когда необходимо зафиксировать срок рассмотрения документа или имя нового типа прав.

## Рекомендации при разработке программного кода

• группируйте программный код функций, структур или констант в отдельные файлы и папки. Например, можно объединить все серверные функции типа сущности OfficialDocument , которые вызываются при подписании документа. Благодаря этому улучшается структурированность кода и упрощается навигация по нему;

• для получения данных вместо SQL-запросов используйте LINQ-запросы ;

• локализуйте интерфейс сразу. Если понадобится менять состав поддерживаемых языков локализации, это поможет сэкономить ресурсы;

• пишите оптимальный код, чтобы действия выполнялись быстро. Подробнее см. в рекомендациях по разработке для масштабируемых систем .
