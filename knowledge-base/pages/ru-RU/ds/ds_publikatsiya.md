# Публикация

Публикация – процесс установки, обновления и удаления решений в локальной или продуктивной системе.

ВАЖНО. Публиковать решения в продуктивную систему рекомендуется в нерабочее время, так как при этом веб-сервер перезагружается и недоступен для пользователей. Если сотрудник работает с объектами системы, которые изменили в обновляемом решении, могут возникнуть конфликты. Чтобы предотвратить это, н а период публикации администратор может включить режим технического обслуживания .

Перед публикацией в продуктивную систему создайте резервную копию БД. Это позволит восстановить данные при возникновении ошибок.

Для публикации решения в систему с существующими пакетами требуется указать полный комплект: базовые прикладные, установленные и новые пакеты.

После публикации в локальную или продуктивную систему проверьте, что изменения не повлияли на работоспособность веб-сервера и сервисов Directum RX. Подробнее см. в разделе «Проверка работоспособности сервисов» ( Windows , Linux ).

## Публикация в локальную систему

Чтобы проверить внесенные изменения, опубликуйте их в локальную систему с помощью среды разработки . Процесс публикации состоит из следующих этапов:

- 1. В среде разработки на основе исходных кодов создается пакет развертывания .
- 2. Пакет развертывания отправляется на веб-сервер.
- 3. В базе данных обновляются структуры, и копируются обновленные файлы модулей.
- 4. Веб-сервер запускает механизм обновления прикладной разработки.
- 5. Если разработчик изменял не только прикладной код и ресурсы, то сервисы перезапускаются. Примеры, когда рестарт не происходит, см в разделе «Публикация с помощью среды разработки».

## Публикация в продуктивную систему

Чтобы внесенные изменения стали доступны сотрудникам, опубликуйте их в продуктивную систему с помощью инструмента Directum Launcher или через командную строку . Процесс публикации состоит из следующих этапов:

- 1. Разработчик создает пакет разработки.
- 2. На основе пакета разработки создается пакет развертывания и отправляется на веб-сервер.
- 3. В базе данных обновляются структуры, и копируются обновленные файлы модулей.
- 4. Веб-сервер запускает механизм обновления прикладной разработки.
- 5. Веб-сервер и сервисы перезапускаются.

## Разница в процессах публикации с помощью среды разработки и Directum Launcher

Предположим, в системе установлено решения 1 и 2. Нужно установить решение 3 и обновить решение 2.

В среде разработки при публикации 2 и 3, установится решение 3 и обновится 2, а 1 удалится.

При публикации с помощью инструмента Directum Launcher в системе появится решение 3, обновится 2 и останется решение 1.

## Механизм обновления прикладной разработки

При обновлении веб-сервер проверяет БД на наличие актуальной версии прикладной разработки. Если обнаруживается новый пакет публикации, веб-сервер получает изменения и помещает их в папку <ServerPath>\bin\AppliedModules. Затем выполняется отложенный автоматический рестарт веб-сервера, после чего новая прикладная разработка загружается на веб-сервер.
