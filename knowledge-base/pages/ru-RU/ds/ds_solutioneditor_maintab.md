# Редактор решения

Чтобы задать или изменить настройки решения, нажмите на его название в дереве решений. Откроется редактор, в котором:

- задаются параметры решения;
- добавляются функции.

При открытии редактора отображается вкладка «Главная», на которой задаются параметры , зависимости перекрытий модулей , функции , структуры и константы .

Именование

*Имя . Указывается при создании решения. Оно должно быть уникальным, состоять из букв латинского алфавита и цифр. *Имя в базе данных . Используется при генерации таблиц для перекрытий в базе данных. Заполняется автоматически. При необходимости имя в базе данных можно изменить. Для этого введите нужное значение вручную. Имя решения должно содержать от 2 до 7 символов, быть уникальным, состоять из букв латинского алфавита и цифр и начинаться с буквы. *Идентификатор . Уникальный ключ решения, который присваивается ему при создании. Используется для однозначного определения объекта и обращения к нему в системе.

Параметры

*Код компании . Используется при написании программного кода и генерации уникальных имен таблиц в базе данных. По коду можно определить, какая компания разработала решение. Заполняется автоматически и доступен только для чтения. *Версия . Номер версии решения. Заполняется автоматически при создании решения. Последняя цифра версии увеличивается на 1 после каждого создания пакета разработки с исполняемыми файлами. Сбросить прикладной кэш при публикации . Установите флажок, если в программном коде изменилась работа с кэшем. Например, изменилось вычисление для виджетов модулей решения.

Зависимости

Зависимости . Список, который состоит из: • модулей базового решения , которые перекрыты в вашем решении; • модулей базового и ваших решений, программный код или элементы разработки которых используются в перекрытиях модулей или функциях вашего решения. ВАЖНО. Добавляйте модуль в список зависимостей сразу после использования в своем решении. Если зависимости не указаны, то при публикации возникнут ошибки . Например, если в своем решении вы перекрыли модуль Sungero.Company, добавьте модуль базового решения в список зависимостей. Для этого: 1. По кнопке добавьте новую строку в таблице модулей. 2. В созданной строке перейдите к колонке Модуль и из выпадающего списка выберите модуль Sungero.Company. 3. В поле Мин. версия автоматически подставится текущая версия выбранного модуля. Рекомендуется оставлять значение по умолчанию. При необходимости заполните поле Макс. версия . При загрузке зависимого модуля проверяется его версия, используемая в решении. Если она не входит в промежуток между значениями, указанными в полях Мин. версия и Макс. версия , модуль не загружается. Если зависимость добавлена по ошибке, перейдите к ней и нажмите на кнопку . Не рекомендуется удалять зависимости из списка, даже если элементы модуля больше не используются.

Функции

Добавьте функции решения. Они могут быть клиентскими, серверными и разделяемыми. Функции типа сущности предназначены для повторного использования кода в нескольких элементах решения, например в перекрытиях модулей. Подробнее см. раздел Подробнее см. раздел «Создание функций».

Структуры и константы
