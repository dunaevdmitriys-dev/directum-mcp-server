# Блок «Уведомление»

Блок типа «Уведомление» (NoticeBlock) предназначен для создания уведомлений. Выход из блока происходит сразу после создания уведомлений.

## Свойства блока

Свойство | Описание
Заголовок ( Title ) | Отображается в схеме задачи
Описание (Description) | Поясняющий текст, который отображается на панели настроек блока на схеме варианта процесса. ПРИМЕЧАНИЕ. В прикладном коде свойство доступно только для чтения
Инициатор (Author) | Инициатор уведомления
Исполнители ( Performers ) | Список исполнителей. Для выбора доступны роли и наследники от групп. совет. Sid (идентификатор субъекта прав) можно узнать в системной таблице Sungero_Core_Recipient
Тема ( Subject ) | Тема создаваемого уведомления
Тема в переписке (ThreadSubject) | Тема уведомления в переписке
Представление по умолчанию (DefaultViewForm) | Представление формы , которое отображается при открытии карточек отдельно от списка. Если свойство не задано, подбирается наиболее подходящее представление
Представление в списке (EmbeddedViewForm) | Представление формы для карточки в списке . Если свойство не задано, отображается представление по умолчанию

## Серверные события блока

Событие | Назначение | Аргументы события
Старт блока (<Имя блока>Start) | Заполнение свойств выполняемого блока. Событие выполняется на веб-сервере при старте обработки блока | _obj – задача, в рамках которой обрабатывается блок. _block – блок, в рамках которого выполняется событие _block.ProcessStagesDisplayMode – режим отображения блока в этапах процесса. Возможные значения: • Show – пока не выполнен. Блок отображается, если он выполняется в текущий момент или в дальнейшем; • ShowInWork – в работе. Блок отображается, если он выполняется в текущий момент; • Hide – никогда. Блок скрыт. Совет. Изменять режим отображения рекомендуется в веб-клиенте через свойство блока Отображать в этапах процесса
Старт уведомления (<Имя блока>StartNotice) | Заполнение свойств уведомления, которые нельзя заполнять через свойства блока. Например, свойства типа Текст , Картинка , Бинарные данные . Событие вызывается до старта уведомления (после создания и заполнения свойств уведомления) и до сохранения в базу данных | _obj – задача, в рамках которой обрабатывается блок. notice – уведомление, созданное в рамках блока. _block – блок, в рамках которого выполняется событие _ block.ProcessStagesDisplayMode – режим отображения блока в этапах процесса. Возможные значения: • Show – пока не выполнен. Блок отображается, если он выполняется в текущий момент или в дальнейшем; • ShowInWork – в работе. Блок отображается, если он выполняется в текущий момент; • Hide – никогда. Блок скрыт. Совет. Изменять режим отображения рекомендуется в веб-клиенте через свойство блока Отображать в этапах процесса
Конец блока (<Имя блока>End) | Актуализация свойств задачи после создания уведомлений. Событие вызывается при завершении обработки блока. ПРИМЕЧАНИЕ. Изменение свойств объектов, заданное в веб-клиенте при завершении блока, происходит после выполнения события «Конец блока» | _obj – задача, в рамках которой обрабатывается блок. _block – блок, в рамках которого выполняется событие. сreatedNotices – уведомления, созданные в рамках блока _block.ProcessStagesDisplayMode – режим отображения блока в этапах процесса. Возможные значения: • Show – пока не выполнен. Блок отображается, если он выполняется в текущий момент или будет выполняться в дальнейшем; • ShowInWork – в работе. Блок отображается, если он выполняется в текущий момент; • Hide – никогда. Блок скрыт. совет. Изменять режим отображения рекомендуется в веб-клиенте через свойство блока Отображать в этапах процесса

## Пример. Назначение исполнителя текущей задачи получателем уведомления

```csharp
public virtual void MyNoticeBlockStart()
{
// Назначить исполнителя текущей задачи получателем уведомления.
_block.Performers.Add(_obj.Performer);
}
```

## Клиентские события блока

Событие | Назначение | Аргументы события
Обновление блока (<Имя блока>RefreshBlock) | Задание логики для отображения свойств блока в зависимости от условий. Вызывается при открытии схемы процесса в редакторе и изменении свойств блока | _block – блок, в рамках которого выполняется событие _block.<название свойства>.Value – значение свойства; _block.<название свойства>.IsEmpty() – проверка, заполнено ли свойство; _block.<название свойства>.IsExpression() – проверка, является ли свойство вычисляемым выражением; _block.State.Properties.<название свойства>.<состояние> – состояние свойства. Возможные значения: • isVisible – видимое; • isEnabled – доступное; • isRequired – обязательное _block.ProcessStagesDisplayMode – режим отображения блока в этапах процесса. Возможные значения: • Show – пока не выполнен. Блок отображается, если он выполняется в текущий момент или будет выполняться в дальнейшем; • ShowInWork – в работе. Блок отображается, если он выполняется в текущий момент; • Hide – никогда. Блок скрыт. Совет. Изменять режим отображения рекомендуется в веб-клиенте через свойство блока Отображать в этапах процесса

## Пример. Отображение выпадающего списка в зависимости от того, установлен ли флажок

```csharp
public virtual void AssignmentBlockRefresh()
{
// Если флажок PropertyBool установлен, то отобразить выпадающий список PropertyEnum
_block.State.Properties.PropertyEnum.IsVisible = _block.PropertyBool.Value.GetValueOrDefault();
}
```
