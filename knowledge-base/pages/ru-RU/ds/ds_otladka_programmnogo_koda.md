# Отладка программного кода

Отладка используется для поиска и устранения ошибок в программном коде, например, если он работает некорректно или возникает исключение. Она позволяет:

- выполнять программный код до точек останова и по шагам;
- просматривать значения переменных и стек вызовов функций.

При работе со средой разработки Development Studio код отлаживается в стороннем редакторе VS Code. Подробнее про отладку см. в документации Visual Studio Code статью Debug code with Visual Studio Code .

Для отладки нужна привилегия «Отладка программ» (SeDebugPrivilege). По умолчанию она есть у локального администратора. Также ее можно выдать отдельно, не включая пользователя в группу администраторов.

Порядок отладки программного кода:

- 1. Откройте нужный файл через среду разработки или сразу в стороннем редакторе кода.
- 2. Нажмите на кнопку отладки.
- 3. Выберите конфигурацию отладки в зависимости от процесса, который нужно отладить. Процессы отвечают за запуск определенного кода:

Конфигурация | Процесс | Что можно отладить
GenericService Attach (workspace) | Общий сервис (Sungero.GenericService.Host) | Схемы задач, асинхронные обработчики, фоновые процессы, отчеты и виджеты Примечание. Отладка общего сервиса возможна, если он выполняет функции четырех сервисов: сервис выполнения блоков задач, сервис асинхронных событий, сервис отчетов, сервис виджетов
WebServer Attach (workspace) | Веб-сервер (Sungero.WebServer.Host) | Клиентский и серверный код, функции инициализации
.NET Core Attach Примечание . Если нет отдельного процесса для отладки, то выберите конфигурацию для общего сервера | Сервис асинхронных событий (Sungero.Worker.Host) | Фоновые процессы , асинхронные обработчики
Сервис обработки схем задач (Sungero.Workflow.Host) | Схемы задач
Сервис отчетов (Sungero.ReportService.Host) | Отчеты
Сервис виджетов (Sungero.Widgets.Host) | Виджеты
IntegrationService Attach (workspace) | Сервис интеграции (Sungero.IntegrationService.Host) | Функции интеграции

- 4. Запустите отладку.

Совет. Для поиска ошибок также можно использовать лог-файлы .
