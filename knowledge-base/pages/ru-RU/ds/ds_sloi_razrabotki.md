# Перекрытие

Механизм перекрытия позволяет на рабочем слое разработки безопасно модифицировать модули и типы сущностей базового решения : изменять логику работы и добавлять новую функциональность.

Например, чтобы изменить логику подписания типа документа базового решения, необходимо создать свое решение, перекрыть в нем необходимый тип документа и переопределить обработчик события Подписание .

Так как перекрытые модули и типы сущностей хранятся на отдельном слое разработки, вы можете вносить в них любые изменения, не нарушая при этом логику работы базового решения. Это значит, что при обновлении Directum RX изменения новой версии безопасно объединяются с вашими изменениями. При этом, если в базовом решении изменится та же часть, которую вы доработали в своем решении, у пользователей будут отображаться только ваши изменения.

Совет. Перед началом разработки перекрытий ознакомьтесь с рекомендациями .

## Создание перекрытия

- 1. Перейдите к решению, в котором необходимо создать перекрытие, выделите пункт « Перекрытые элементы » .
- 2. По кнопке раскройте выпадающий список и выберите модуль или тип сущности, для которого создается перекрытие, например например, Документ :

3. В открывшемся окне в выпадающем списке выберите перекрываемый модуль или тип сущности и нажмите на кнопку Перекрыть :

- В результате в дереве решений в разделе «Перекрытые элементы» появляется элемент и открывается его редактор:
- 4. Если вы перекрывали модуль, добавьте его в список зависимостей решения.
- 5. Внесите необходимые изменения. Подробнее см. раздел «Что можно изменять в перекрытии» .

## Перекрытие абстрактных типов сущностей

Если необходимо внести идентичные изменения в существующие типы сущностей, созданные на основе одних и тех же родительских типов, перекройте базовые типы. Например, чтобы изменить логику создания всех договорных документов, перекройте тип документа ContractualDocument и переопределите нужный обработчик события.

Предположим, необходимо изменить логику сохранения всех договорных документов. Для этого:

- 1. Создайте решение.
- 2. Перекройте тип сущности Договорной документ ( ContractualDocument ), так как он является предком для всех договоров.

Совет. Чтобы найти предка типа сущности, откройте его редактор и перейдите к параметру Наследовать от . В нем указано имя типа сущности, от которого наследуются свойства, действия, логика работы и др.:

- 3. В редакторе перекрытого базового типа документа переопределите обработчик нужного события.

В итоге у всех наследников договорного документа ( ContractualDocument ) изменяется логика выполнения обработчика события. При этом перекрытый базовый тип сущности находится на рабочем слое, а у конечных типов сущностей изменяется цепочка наследования. Перекрытый базовый тип документа становится предком для измененных наследников:

В результате при создании типа документа перекрытый тип можно выбрать в качестве базового типа сущности. При этом базовый тип сущности, который перекрывается, выбрать нельзя. Так в примере выше в перекрытие изменяется обработчик договорного документа ( ContractualDocument ), в дальнейшем при создании нового типа договорного документа в качестве родительского типа можно выбрать только перекрытие базового типа. При этом у создаваемых типов документов по умолчанию будет измененный обработчик события.
