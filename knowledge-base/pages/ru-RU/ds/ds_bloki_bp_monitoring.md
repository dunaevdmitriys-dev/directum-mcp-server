# Блок «Мониторинг»

Блок типа «Мониторинг» (MonitoringBlock) предназначен для контроля наступления определенного события. Например, можно использовать блок, чтобы отправлять уведомление, когда у документа изменяется свойство.

## Свойства блока

Свойство | Описание
Заголовок ( Title ) | Отображается в схеме задачи
Описание (Description) | Поясняющий текст, который отображается на панели настроек блока на схеме варианта процесса. ПРИМЕЧАНИЕ . В прикладном коде свойство доступно только для чтения
Период ( Period ) | Интервал времени, через который выполняется мониторинг
Срок прекращения ( Deadline ) | Срок принудительного завершения мониторинга относительно даты и времени старта

## Выходные свойства блока

Свойство | Описание
Результат выполнения (ExecutionResult) | Результаты выполнения блока. Значения по умолчанию: Success (Успешно), Deadline (Срок истек). <Результат выполнения> – произвольный результат, добавленный разработчиком

## Серверные события блока

Событие | Назначение | Аргументы события | Возвращаемое значение
Старт блока (<Имя блока>Start) | Заполнение свойств блока. Событие вызывается при старте обработки блока | _obj – задача, в рамках которой обрабатывается блок. _block – блок, в рамках которого выполняется событие _block.ProcessStagesDisplayMode – режим отображения блока в этапах процесса. Возможные значения: • Show – пока не выполнен. Блок отображается, если он выполняется в текущий момент или в дальнейшем; • ShowInWork – в работе. Блок отображается, если он выполняется в текущий момент; • Hide – никогда. Блок скрыт. Совет. Изменять режим отображения рекомендуется в веб-клиенте через свойство блока Отображать в этапах процесса
Результат мониторинга (<Имя блока>Result) | Определение завершения мониторинга. Событие вызывается при каждом выполнении мониторинга | _obj – задача, в рамках которой обрабатывается блок _block.ProcessStagesDisplayMode – режим отображения блока в этапах процесса. Возможные значения: • Show – пока не выполнен. Блок отображается, если он выполняется в текущий момент или в дальнейшем; • ShowInWork – в работе. Блок отображается, если он выполняется в текущий момент; • Hide – никогда. Блок скрыт. Совет. Изменять режим отображения рекомендуется в веб-клиенте через свойство блока Отображать в этапах процесса | Результат мониторинга. Возможные значения: True . Ожидаемое событие наступило, переход по стрелке Успешно . Мониторинг завершится. False . Ожидаемое событие не наступило. В этом случае проверяется свойство Срок прекращения (RelativeDeadline), если время, указанное в свойстве: • еще не наступило , то вычисляется, что ближе по времени: следующий период мониторинга или срок прекращения. Событие будет запущено еще раз через заданный период, при наступлении срока прекращения или при вызове события из кода; • наступило , то происходит переход по стрелке Срок истек . Мониторинг завершается
Конец блока (<Имя блока>End) | Асинхронное выполнение операции при завершении блока. Событие выполняется при завершении обработки блока. ПРИМЕЧАНИЕ. Изменение свойств объектов, заданное в веб-клиенте при завершении блока , происходит после выполнения события «Конец блока» | _obj – задача, в рамках которой обрабатывается блок. _block – блок, в рамках которого выполняется событие _block.ProcessStagesDisplayMode – режим отображения блока в этапах процесса. Возможные значения: • Show – пока не выполнен. Блок отображается, если он выполняется в текущий момент или в дальнейшем; • ShowInWork – в работе. Блок отображается, если он выполняется в текущий момент; • Hide – никогда. Блок скрыт. Совет. Изменять режим отображения рекомендуется в веб-клиенте через свойство блока Отображать в этапах процесса

## Пример. Результат мониторинга в зависимости от выполнения всех заданий по текущей задаче

```csharp
// Результат мониторинга. Если все задания по текущей задаче выполнены, то
// мониторинг завершится с результатом true, иначе – false.
public virtual bool MyMonitoringBlockResult()
{
// Поиск заданий по текущей задаче.
var assignments = Sungero.Workflow.Assignments.GetAll()
.Where(a => a.MainTask == _obj.Id);
return assignments.All(a => a.Status == Sungero.Workflow.Assignment.Status.Completed);
}
```

## Клиентские события блока

Событие | Назначение | Аргументы события
Обновление блока (<Имя блока>RefreshBlock) | Задание логики для отображения свойств блока в зависимости от условий. Выполняется при открытии схемы процесса в редакторе и изменении свойств блока | _block – блок, в рамках которого выполняется событие. _block.<название свойства>.Value – значение свойства. _block.<название свойства>.IsEmpty() – проверить, заполнено ли свойство. _block.<название свойства>.IsExpression() – проверить, является ли свойство вычисляемым выражением. _block.State.Properties.<название свойства>.<состояние> – состояние свойства. Возможные значения: • isVisible – видимое; • isEnabled – доступное; • isRequired – обязательное _block.ProcessStagesDisplayMode – режим отображения блока в этапах процесса. Возможные значения: • Show – пока не выполнен. Блок отображается, если он выполняется в текущий момент или будет выполняться в дальнейшем; • ShowInWork – в работе. Блок отображается, если он выполняется в текущий момент; • Hide – никогда. Блок скрыт. Совет. Изменять режим отображения рекомендуется в веб-клиенте через свойство блока Отображать в этапах процесса

## Пример. Отображение выпадающего списка в зависимости от того, установлен ли флажок

```csharp
public virtual void AssignmentBlockRefresh()
{
// Если флажок PropertyBool установлен, то отобразить выпадающий список PropertyEnum
_block.State.Properties.PropertyEnum.IsVisible = _block.PropertyBool.Value.GetValueOrDefault();
}
```
