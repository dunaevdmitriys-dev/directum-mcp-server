# Рекомендации по настройке панели фильтрации

Для удобного использования и поддержания единого стиля системы при настройке панели фильтрации соблюдайте рекомендации:

- выбирайте типы контролов в зависимости от решаемой задачи;
- используйте стандартную группировку контролов ;
- выносите на панель значения , по которым записи фильтруются чаще всего.

## Типы контрола

Период дат

Для удобной фильтрации по датам рекомендуется добавить предопределенные периоды, наиболее подходящие контексту списка, и возможность ввода произвольного периода. При этом используйте контрол «Период дат» вместе с контролом «Набор флажков». В этом случае флажки набора соответствуют наиболее частым периодам, по которым фильтруются сущности в списке: последние 7, 30, 90 дней и др. А контрол «Период дат» становится доступным, если установлен определенный флажок, например, Произвольный период:

Рекомендуемые предопределенные периоды:

- для большинства типов документов в качестве периодов используются последние 7, 30 и 90 дней. Они позволяют вывести недавние объекты за постоянный период;
- для учетных документов следует использовать календарные периоды: текущий месяц, предыдущий месяц, текущий квартал, предыдущий квартал и т.п. Такие периоды часто используются в бухгалтерском учете;
- для задач и заданий используйте периоды 30, 90 и 180 дней.

СОВЕТ. Оставляйте возможность указывать произвольный период дат, если это не критично для скорости открытия списка. Например, в папках потоков входящих заданий для ускорения их открытия такая фильтрация отключена.

Набор флажков

Контрол может отображаться в виде:

- переключателя – для выбора одного значения. Используйте, если критерии фильтрации взаимно исключают друг друга;
- набора флажков – для выбора нескольких значений одновременно. Используйте, например, если у свойства сущности есть несколько значений и нужно выбирать их в различных комбинациях.

Переключатель | Набор флажков

Для флажков рекомендуется выбирать такие значения, чтобы они покрывали все возможные значения связанного критерия. Если их много, то выделите 3-4 часто используемых варианта, а остальные отнесите к флажку Прочие :

При написании обработчика фильтрации рекомендуется учитывать выбранные флажки по логическому условию «ИЛИ». При этом, если не выбран ни один флажок, условие по этому набору флажков накладывать не нужно. Например:

```csharp
// Если выбрано хоть одно значение, то искать записи, в которых встречается
// хотя бы один выбранный статус.
if (_filter.Registered || _filter.Reserved || _filter.NotRegistered)
query = query.Where(l =>
(_filter.Registered && l.RegistrationState ==
OfficialDocument.RegistrationState.Registered) ||
(_filter.Reserved && l.RegistrationState ==
OfficialDocument.RegistrationState.Reserved) ||
(_filter.NotRegistered && l.RegistrationState ==
OfficialDocument.RegistrationState.NotRegistered));
```

Ссылка на сущность

Контрол используется для выбора сущности заданного типа. Может отображаться в виде:

- выпадающего списка;
- отдельной формы-списка, которая открывается при нажатии на кнопку .

Выпадающий список | Форма-список

Рекомендуется скрывать значения, которые неактуальны для текущего списка. Например, в списке «Реестр договоров» можно выбрать только договорные виды документов:

```csharp
// Показать только договорные виды документов.
var query = query
.Where(d => d.Status == CoreEntities.DatabookEntry.Status.Active)
.Where(d => d.DocumentFlow == Docflow.DocumentKind.DocumentFlow.Contracts);
```

## Группировка контролов

Для панелей фильтрации рекомендуется использовать стандартную группировку контролов:

- в верхней части – фильтры по состоянию или по типам сущностей списка;
- в нижней части – фильтры по датам;
- в средней части – остальные контролы, расположенные в порядке убывания важности.

СОВЕТ. Добавляйте на панель только часто используемые фильтры, чтобы не появлялась вертикальная полоса прокрутки.

Располагайте однотипные элементы панели фильтрации одинаково. Например, в списках входящих и исходящих документов есть группа для фильтрации по интервалу времени. В этом случае группа должна выглядеть однотипно и находиться в одном и том же месте панели фильтрации в обоих списках.

При написании обработчика фильтрации объединяйте группы между собой по логическому условию «И», т.е. складывайте условия. Например, наложение фильтров используется в списке «Входящие счета»:

```csharp
// Фильтрация по состояниям.
if ((_filter.Draft || _filter.OnApproval || _filter.PayAccepted || _filter.PayRejected) &&
!(_filter.Draft && _filter.OnApproval && _filter.PayAccepted && _filter.PayRejected))
...
// Фильтрация по группе "Основные" накладывается к предыдущим фильтрам.
if (_filter.Counterparty != null)
query = query.Where(x => Equals(x.Counterparty, _filter.Counterparty));
...
// Фильтрация по датам накладывается к предыдущим фильтрам.
if (_filter.Last7daysInvoice)
query = query.Where(c => (c.Date ?? c.Created) >= Calendar.Today.AddDays(-7));
```

## Значения критериев

Выносите на панель значения, по которым записи фильтруются чаще всего. Например, если необходимо оставить только входящие письма, зарегистрированные в конкретном журнале регистрации, в этом случае на панель фильтрации выносятся свойства Вид документа и Журнал регистрации .

Для панели фильтрации у контролов рекомендуется устанавливать значения по умолчанию:

- период – минимальный период среди предопределенных;
- состояние записей – Действующие , если есть фильтрация по действующим и закрытым записям;
- флажки в группах – в большинстве случаев сняты;
- представление списка – основное, редко используемые записи скрыты. Например, по умолчанию в списке заданий показываются только задания в работе.

Рекомендации для панели фильтрации в папке потока:

- добавьте группу с фильтрацией заданий по состоянию: В работе , Все . Значение по умолчанию В работе ;
- добавьте фильтрацию по дате создания с тремя вариантами: 30 дней , 90 дней , 180 дней . Значение по умолчанию 90 дней . Это уменьшает размер выборки и исключает из нее старые задания. Такое ограничение ускоряет открытие списка и обновление количества входящих заданий;
- если выбрана фильтрация по состоянию В работе , то период дат сделайте недоступным для изменения и не учитывайте его при формировании выборки.
