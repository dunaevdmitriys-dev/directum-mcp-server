# Создание функций модуля

Предположим, что для модуля необходимо создать серверную функцию, после этого нужно создать клиентскую функцию и вызвать из нее серверную. Для этого:

- 1. В редакторе модуля перейдите на вкладку «Главная».
- 2. В группе «Функции» рядом со ссылкой Серверные нажмите на кнопку .
- 3. В открывшемся редакторе кода измените имя созданной функции, добавьте комментарий с описанием и напишите ее код. При необходимости измените тип возвращаемого значения и добавьте входные параметры, модификаторы и атрибуты функции .
- Пример кода:

```csharp
/// <summary>
/// Создать организацию.
/// </summary>
/// <returns>Организация.</returns>
[Remote]
public static ICompany CreateCompany()
{
return Companies.Create();
}
```

В примере кода используется атрибут Remote , так как серверная функция вызывается в клиентском коде.

- важно. Если серверная функция будет использоваться в выражении объекта отчета, то в объявлении функции явно укажите модификатор static .

4. Сохраните изменения.

- 5. В Development Studio в группе «Функции» рядом со ссылкой Клиентские нажмите на кнопку .
- 6. В открывшемся редакторе кода доработайте имя клиентской функции, добавьте комментарий с описанием и вызов серверной функции.
- Чтобы вызвать серверную функцию используйте формат:
- Functions.Module.Remote.<Имя функции>(<Параметры>) . В формате используется Remote , потому что из клиентского кода вызывается серверная функция. Формат вызова других функций ниже :

```csharp
/// <summary>
/// Клиентская функция для создания организации.
/// </summary>
public static void CreateCompany()
{
Functions.Module.Remote.CreateCompany().Show();
}
```

## Формат вызова функций модуля

Функции вызываются по формату Functions.Module.<Имя функции>(<Параметры>) . При этом вызов функции с атрибутом:

- Remote , то есть когда серверная функция вызывается в клиентском коде, происходит в формате:
- Functions.Module. Remote .<Имя функции>(<Параметры>)
- Public , то есть когда функция вызывается в функции другого модуля, происходит в формате:
- Functions.Module. PublicFunctions .<Имя сущности>.<Имя функции>(<Параметры>)

Формат вызова можно сократить, если функция вызывается:

- из функции типа сущности. Functions можно не указывать;
- из функции того же модуля, в котором она создана. Functions.Module можно не указывать.
