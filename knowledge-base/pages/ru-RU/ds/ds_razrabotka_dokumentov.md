# Разработка документов

Документ в системе представляет собой карточку и содержимое, для которого может быть создано несколько версий. Документы можно создавать , отправлять на согласование, регистрировать, сравнивать, подписывать и многое другое.

## Особенности разработки

Подробнее о порядке создания типа документа см. в разделе «Типы сущностей» . Учитывайте особенности:

- у документа может быть содержимое (Body) с версиями (Versions) ;
- документы можно связывать между собой (Relations) ;
- содержимое документов можно подписывать (Signatures) ;
- для дополнительной защиты можно шифровать документы, включать для них строгий доступ , а также ограничивать работу с ними вне системы ;
- при создании типа документа от базового типа Sungero.Content.ElectronicDocuments наследуются основные свойства и методы , действия и операции авторизации.

Так как при создании типов документов есть особенности, то для них появляются дополнительные настройки в карточке и методы, которые обеспечивают возможность программного экспорта , импорта , работу с версиями, электронными подписями и т.д.

Какие типы документов можно перекрывать , а от каких – создавать наследников , можно посмотреть в среде разработки. При создании типа документа правильно выбирайте базовый тип , от которого наследуется логика работы. Например, нужно создать «Договор закупки», в нем должен поддерживаться механизм регистрации документов и особенности по работе с договорами. В этом случае в качестве базового типа сущности укажите ContractBase – базовый договор. В нем есть необходимые для договора свойства и логика, а также он наследует механизм регистрации документов от OfficialDocument . Подробнее см. схему наследования договорных документов.

Важно . Задачей на согласование по регламенту можно отправлять только документы, которые являются наследниками OfficialDocument .

## Примеры разработки

Используйте примеры разработки, которые помогут изучить возможности среды разработки и подскажут идеи для развития модификации системы:

- Документы
- Разработка модуля и типа документа
- Интеллектуальная обработка документов
- Доступность, обязательность, видимость свойств

## События типа документа

Для типа документа кроме основных серверных и клиентских событий дополнительно доступны события:

Серверные события | Разделяемые события
До подписания (BeforeSigning) | До показа диалога подписания (ShowingSignDialog)
Смена типа (ConvertingFrom)

Подробнее о порядке выполнения событий при подписании см. в разделе «Подписание документа» . Также в каждом типе сущности можно задать события свойств и действий .

## Объектная модель для работы с типами документов

Подробнее об объектной модели для работы с типами документов см. в разделах:

- ElectronicDocuments – документы
- Versions – работа с версиями
- Relations – связывание документов
- ElectronicDocumentTemplates – шаблоны документов
- AssociatedApplications – приложения-обработчики
- Signatures – подписи
- Базовое решение Directum RX
- Иерархия документов
