# Подписание

Подписание документов выполняется на сервисе NOMAD. При этом подписываются основной документ и его приложения, которые вложены в задание на подписание.

По умолчанию документы подписываются простой ЭП без использования сертификата ЭП сотрудника, даже если он добавлен в Solo или Jazz.

Подписание усиленной ЭП возможно в заданиях на согласование по регламенту, если в конфигурации для варианта выполнения задания добавлен атрибут checkStrongSign . В этом случае сервис NOMAD проверяет, нужно ли подписывать документ с использованием сертификата на данном этапе согласования. Если подпись требуется, на документы ставится подпись с типом, указанным в атрибуте checkStrongSign : согласующая или утверждающая.

Пример настройки подписания усиленной ЭП при согласовании официального документа руководителем :

```csharp
<job type="Sungero.Docflow.IApprovalManagerAssignment">
<result name="Approved" checkStrongSign="Endorsing" />
</job>
```

Больше примеров по адаптации сервиса NOMAD для собственных доработок по подписанию документов см. в разделе «Примеры доработок в файле SungeroAdapter.config» .
