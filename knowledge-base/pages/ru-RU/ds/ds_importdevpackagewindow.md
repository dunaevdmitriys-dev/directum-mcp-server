# Импорт разработки

ПРИМЕЧАНИЕ. В текущей версии импорт пакета разработки поддерживается только с помощью старой среды Directum RX Development Studio.

При импорте пакета в репозитории обновляется прикладная разработка. В результате на базовом и рабочем слоях можно использовать актуальную версию решений.

Для этого:

- 1. Откройте старую среду разработки.
- 2. В главном меню среды разработки последовательно выберите пункты Сборка , Импортировать разработку :
- 3. В открывшемся окне в поле Пакет разработки укажите путь к файлу пакета разработки с исходными кодами. Если исходных кодов в пакете нет, то при импорте появляется соответствующее сообщение.
- 4. Определите состав решений, которые нужно перенести в среду разработки. Установите рядом с ними флажки. Если решения зависят от других решений, установите флажки и для них.
- 5. Нажмите на кнопку Импорт .

В результате:

- исходные коды из пакета разработки сохраняются в локальном репозитории ;
- импортированные решения появляются в дереве решений среды разработки.

## Особенности импорта базовых решений

Чтобы после импорта пакета разработки с базовыми решениями изменения появились в локальной системе , обновите ее. Для этого в новой среде разработки:

- 1. Опубликуйте решения.
- 2. Инициализируйте модули.
- 3. Примените настройки по умолчанию для бизнес-процессов и интерфейса. Для этого на панели инструментов среды разработки нажмите на кнопку и в выпадающем списке выберите пункт Применение настроек :

В результате разработка и настройки обновляются. При этом:

- если вычисляемая роль по умолчанию уже существует, то при применении настроек она не перезаписывается;
- если в пакете есть закрытые настройки по умолчанию, то при импорте они закрываются и в системе. При этом закрытая запись не создается, если в системе нет соответствующей настройки;
- если импортируется несколько базовых решений, в которых перекрыты одни и те же типы сущностей или модули, то изменения в перекрытиях объединятся .
