# Изолированные области

Изолированные области используются для выполнения кода сторонней сборки в отдельном процессе независимо от других сервисов. Это обеспечивает изоляцию кода : ошибки в стороннем коде не затрагивают стабильность системы, так как он выполняется в отдельном процессе.

Также в изолированную область можно добавлять код, который содержит ресурсоемкие вычисления, например конвертацию больших документов. Это снижает нагрузку на основной процесс и обеспечивает стабильность работы системы.

## Создание изолированной области

- 1. В редакторе модуля на вкладке «Изолированные области» нажмите на кнопку :
- 1. В открывшемся окне укажите имя изолированной области. Оно должно быть уникальным, состоять из букв латинского алфавита, цифр и начинаться с буквы.
- 2. Перейдите к коду области и функций . Для этого перейдите по ссылкам «Код области» или «Функции» соответственно. Чтобы создать функцию, нажмите на кнопку рядом с ссылкой Функции . Задайте логику выполнения для Кода области .

При создании изолированной области учитывайте ее особенности и ограничения .

После сборки решения для каждого модуля создается сборка <Имя модуля>.Isolated.dll .

Примечание. Администратор при необходимости может изменить настройки запуска сервиса изолированного кода при необходимости.

## Функции

В качестве аргументов и результатов выполнения функции используются простые типы, потоки данных базового типа Stream(System.IO.Stream) , списки, словари и изолированные структуры добавленные в этом модуле.

Чтобы структура стала изолированной и доступной в изолированной области, перед ее описанием нужно добавить атрибут Public с параметром Isolated со значением true , например:

```csharp
[Public(Isolated=true)]
partial class DpadSignaturInfo
{
public int SignIcon { get; set; }
public string Status { get; set; }
}
```

Функция вызывается в формате:

Sungero.<Имя модуля>.IsolatedFunctions.<Имя изолированной области>.<Имя метода>(<Список аргументов>)

Чтобы функция стала доступна в клиентском, серверном и разделяемом коде, ее нужно отметить атрибутом Public .

Если в функции используются потоки данных, то нужно учесть их особенности. В качестве возвращаемого значения и аргументов функции можно использовать только потоки данных базового типа Stream(System.IO.Stream) . При этом в коде функции можно использовать любые потоки, например FileStream или MemoryStream .

Передавать можно несколько потоков, а возвращать только один. Возвращенный поток из изолированной функции может быть прочитан один раз, повторная попытка чтения приводит к ошибке.

Пример. Преобразование документа в PDF

```csharp
public class IsolatedFunctions
{
/// <summary>
/// Преобразовать документ в pdf.
/// </summary>
/// <param name="inputStream">Поток с исходным документом.</param>
/// <param name="saveParameters">Опции сохранения.</param>
/// <returns>Поток для записи результата.</returns>
/// <summary>
[Public]
public virtual Stream ConvertPdfToPdf()
{
var resultStream = new MemoryStream();
try
{
var document = new Aspose.Pdf.Document(inputStream);
this.PreprocessPdfDocument(document);
this.Validate(document);
var pdfFormat = this.GetPdfFormat(document, saveParameters);
this.SaveDocument(document, resultStream, pdfFormat);
}
return resultStream;
}
}
```

## Код области

Код области используется для того, чтобы разделять бизнес-логику и ее реализацию в виде функций. В нем можно задавать: классы, структуры, интерфейсы, перечисления и делегаты. Кроме того, в коде можно вызывать как собственные функции, созданные в изолированной области, так и функции из сторонних сборок. Благодаря этому код изолированных функций не содержит дополнительные объявления, что облегчает навигацию по их списку.

Пример. Добавление изображения на страницу PDF-документа

```csharp
{
/// <summary>
/// Класс для преобразования документа.
/// </summary>
public class ConverterBase
{
/// <summary>
/// Добавить изображение на страницу PDF-документа.
/// </summary>
/// <param name="documentPage">Страница документа.</param>
/// <param name="imageStream">Поток с изображением.</param>
protected virtual void AddImageToPage(Page documentPage, Stream imageStream)
{
var image = new Aspose.Pdf.Image() { ImageStream = imageStream };
documentPage.PageInfo = image.BitmapSize.Width > image.BitmapSize.Height ?
this.GetLandscapePageInfo(image.BitmapSize.Width, image.BitmapSize.Height) :
this.GetPageInfo(image.BitmapSize.Width, image.BitmapSize.Height);
this.ResizeImage(image);
this.SetImageProperties(image);
documentPage.Paragraphs.Add(image);
}
}
}
```

## Особенности изолированных областей

- модули с изолированными областями можно перекрывать;
- в изолированной области перекрытого модуля доступны только его изолированные структуры и структуры предка;
- в коде изолированной области нельзя работать с сущностями, репозиториями, базой данных. Данные передаются через параметры функций изолированной области;
- в перекрытых модулях нельзя добавлять новые изолированные области;
- при изменении кода области для публикации разработки требуется перезапуск веб-сервера;
- сервис асинхронных событий (Worker), работающий под .NET Framework, несовместим с изолированными областями. При обращении к их функциям возникает ошибка «Isolated area service is not supported». Для работы с такими функциями используйте сервис под NET Core.

Примечание. Сервис на .NET Framework, рекомендуется применять только для фоновых процессов и асинхронных обработчиков , которые не работают под .NET Core. Например, при использовании устаревших сторонних библиотек;

- по умолчанию доступны библиотеки Aspose, Newtonsoft.Json и другие, подключенные в стандартной разработке. Чтобы использовать их возможности, достаточно в программном коде добавить нужные пространства имен.
