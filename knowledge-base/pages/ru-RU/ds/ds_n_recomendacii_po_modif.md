# Особенности разработки для мобильных решений

Чтобы модифицировать стандартную функциональность системы Directum RX под конкретные задачи организации, можно доработать существующие, а также создать собственные объекты: задачи, задания, процессы и др. Для этого используется среда разработки Development Studio .

Чтобы поддержать собственную разработку в мобильных приложениях Directum Solo и Directum Jazz:

- 1. Ознакомьтесь с особенностями поддержки изменений на сервисе NOMAD:

• типы задач и заданий ;

• карточки документов и справочников .

- Учитывайте их при редактировании конфигурационного файла SungeroAdapter.config.
- 2. Настройте работу мобильных решений с задачами и заданиями в конфигурационном файле SungeroAdapter.config.
- Примеры по адаптации сервиса NOMAD к собственным доработкам системы см. в разделе «Примеры доработок в файле SungeroAdapter.config» .

Совет . Чтобы определить, в контексте какого клиентского приложения выполняется прикладной код, используйте свойство ApplicationType .

Перед запуском сервиса NOMAD проверьте его работу с измененными настройками на тестовой базе. Корректность настроек, которые изменялись в конфигурационных файлах NOMAD, проверяется автоматически. Подробнее см. раздел «Настройка сервиса NOMAD».
