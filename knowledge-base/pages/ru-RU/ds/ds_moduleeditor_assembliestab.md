# Сторонние библиотеки

Для расширения возможностей платформы Directum RX в решения и модули можно добавлять сторонние библиотеки . Для этого:

- 1. Реализуйте необходимую функциональность в виде отдельной .NET-библиотеки. Подробнее см. в документации Microsoft .
- ВАЖНО. Сторонние библиотеки для прикладной разработки Directum RX должны быть собраны под .NET Standard 2.0.
- 2. В репозитории перейдите в папку <Имя решения/модуля>.Libraries. Например, SolutionDescipline\SolutionDescipline.Libraries или CompanyDEV\CompanyDEV.Libraries.
- 3. Добавьте папку с файлами сторонней библиотеки. Ее имя должно быть уникальным, состоять из букв латинского алфавита и цифр.
- 4. Если среда разработки Development Studio не запущена, то запустите ее, чтобы обновить информацию о добавленной сторонней библиотеке.

В результате сторонние сборки , которые входят в состав сторонней библиотеки, можно использовать в программном коде.

ВАЖНО. Сторонняя библиотека может использоваться только в рамках того решения или модуля, в который она добавлена. Чтобы использовать библиотеку в нескольких модулях или решениях, в их папках <Имя решения/модуля>.Libraries добавьте папку сторонней библиотеки с одинаковым набором файлов и сборок одной версии.

Чтобы обезопасить систему от ошибок и нестабильности стороннего кода, его можно перенести на сервис изолированного кода . Для этого подключите стороннюю библиотеку к изолированной области . После этого код сторонней сборки выполняется в отдельном процессе. В результате работа других сервисов продолжается, даже если в стороннем коде возникают ошибки.

Для удаления сторонней библиотеки удалите ее папку из <Имя решения/модуля>.Libraries и запустите среду разработки, чтобы обновить информацию об удаленной библиотеке.
