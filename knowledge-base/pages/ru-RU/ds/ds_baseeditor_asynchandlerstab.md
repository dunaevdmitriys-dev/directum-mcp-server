# Асинхронные обработчики

Асинхронные обработчики позволяют:

- вынести длительные, ресурсоемкие операции, не требующие немедленного возврата результата, с веб-сервера на отдельный сервис Worker ;
- одновременно выполнить вычисления в несколько потоков.

Асинхронные обработчики могут использоваться в Directum RX, например чтобы автоматически выдать сотрудникам права на большое количество документов по настроенным правилам . В этом случае результат не требуется получить сразу же после выполнения (в основном потоке), процесс является ресурсоемким и его можно распараллелить на несколько потоков. Поэтому вычисления удобно выполнить на отдельном сервисе Worker .

Благодаря ним можно ускорить выполнение тяжелых операций, снизить нагрузку на веб-сервер, а также ускорить отклик Directum RX для конечного пользователя.

## Схема работы

- 1. Прикладной разработчик создает асинхронный обработчик в нужном месте серверного кода, заполняет параметры и вызывает его. При этом выполнение серверного кода продолжается.
- 2. Асинхронный обработчик отправляется на сервис Worker .
- 3. На сервисе Worker , то есть за рамками основного потока, асинхронный обработчик выполняет свои вычисления. Если во время его выполнения возникает ошибка, то обработчик повторно запускается через время, заданное при его создании.

## Создание асинхронного обработчика

- 1. В редакторе модуля перейдите на вкладку «Асинхронные обработчики».
- 2. Нажмите на кнопку . В результате создается асинхронный обработчик и открывается его редактор:
- 3. На вкладке «Основное» настройте именование , параметры и свойства .
- 4. На вкладке «Обработчики» задайте логику асинхронного обработчика.
- 5. Вызовите асинхронный обработчик в нужном месте серверного кода по формату <Имя модуля>.AsyncHandlers.<Имя асинхронного обработчика>.Create().
- Пример. Создание и вызов асинхронного обработчика (автоматическая выдача прав)

```csharp
public override void AfterSave(Sungero.Domain.AfterSaveEventArgs e)
{
// Создать асинхронный обработчик GrantAccessRightsToDocument.
var asyncRightsHandler = Docflow.AsyncHandlers.GrantAccessRightsToDocument.Create();

// Заполнить параметры асинхронного обработчика.
asyncRightsHandler.DocumentId = _obj.Id;

// Вызвать асинхронный обработчик.
asyncRightsHandler.ExecuteAsync();
}
```

- При вызове асинхронного обработчика можно указать уведомления , которые нужно показывать пользователям при запуске и завершении выполнения обработчика.
- 6. При необходимости выполните отладку кода асинхронного обработчика.

## Изменение асинхронного обработчика

Чтобы изменить асинхронный обработчик стандартного решения Directum RX, перекройте модуль, к которому он относится. В перекрытии модуля вы сможете изменить интервал повтора, а также обработчик события Выполнение .
