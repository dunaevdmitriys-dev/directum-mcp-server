# Разработка задач и заданий

Для автоматизации бизнес-процессов компании предназначены задачи. В зависимости от потребностей организации может понадобиться создание собственного вида задачи. Это можно сделать в среде разработки, задав необходимые параметры.С помощью подхода no-code для каждого типа задачи разрабатывается схема , при выполнении которой создаются задачи, задания, уведомления и задание на приемку.

Примечание. Для типов задач, схема которых настраивалась в предыдущей версии среды разработки, в редакторе типа сущности вы можете установить флажок Схема настраивается в проводнике . В таком случае для них в веб-клиенте необходимо создать вариант процесса и настроить схему. После установки флажка снять его нельзя.

Карточка задачи содержит:

- закладки для просмотра и редактирования свойств;
- область переписки . Позволяет проследить ход работ по задаче. В области переписки в задаче текст появится после ее старта;
- область вложений . Содержит ссылки на сущности, с которыми работают пользователи. Например, во вложения добавляется документ, который необходимо согласовать и зарегистрировать;
- панель предпросмотра вложений , на которой отображается содержимое вложенного документа.

## Разработка типа задачи

1. Создайте тип задачи для бизнес-процесса, который цифровизируется. Подробнее см. раздел «Типы сущностей» . 2. Создайте свои блоки , с помощью которых состоится процесс. 3. В проводнике веб-клиента настройте бизнес-процесс и его схему. ПРИМЕЧАНИЕ. Если для схемы задачи разрабатывается несколько вариантов процессов, подходящих по критериям, и нужно дать возможность пользователю вручную указывать процесс, свойство «Процесс» ( ProcessKind ) можно вынести на форму с помощью подхода no-code. 4. При необходимости добавьте группы вложений . В них добавляются ссылки на документы, записи справочников, другие задачи. Какие типы задач можно перекрывать , а от каких – создавать наследников , можно посмотреть в среде разработки. При создании типа задачи от базового типа Sungero.Workflow.Task наследуются основные свойства, методы и действия . ПРИМЕЧАНИЕ. В Directum RX нельзя создавать абстрактные типы задач, заданий, уведомлений и заданий на приемку.

## Подчиненные типы сущности задачи

У типа задачи есть подчиненные типы сущностей: • задание для выполнения работ по задаче. В его карточке указаны описание необходимых работ, срок задания и результаты выполнения . По одной задаче можно создавать несколько типов заданий; • уведомление для оповещения пользователей о тех или иных работах. В уведомлении нет результатов выполнения. В рамках одной задачи можно создавать несколько типов уведомлений; • задание на приемку выполненных работ. В его карточке указаны срок и результат приемки работ. Создается только в рамках задачи и не существует за ее пределами. Задание на приемку выполняется с одним из результатов: • Принять , если результаты выполнения устраивают. Задача не завершается до тех пор, пока задание на приемку не выполнено с указанным результатом; • Отправить на доработку , если есть пожелания по доработке результатов. В этом случае задача стартует заново. Примечание. В задаче может быть только одно задание на приемку. Чтобы создать подчиненный тип сущности, по кнопке раскройте выпадающий список и выберите тип сущности: Примечание. Вы можете создавать новые типы заданий и уведомлений в модуле без привязки к типу задачи. В таком случае их можно использовать в разных типах задач модуля, но в них недоступны группы вложений и уникальные свойства, реализованные для конкретного типа задачи. В обработчиках событий таких заданий и уведомлений можно обращаться только к свойствам и группам вложений, реализованным на уровне базовой задачи ( Sungero.Workflow.Task ).

## Объектная модель для работы с задачами и заданиями

• Task – задачи • SimpleTask – простые задачи • Assignment, Notice – задания и уведомления • Attachments – работа со вложениями • Scheme – схема варианта процесса • Blocks – блоки схемы • Базовое решение Directum RX
