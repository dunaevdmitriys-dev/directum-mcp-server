# Логика «Ведущий – ведомый» в свойствах

В среде разработки для отражения логических связей между свойствами предусмотрена возможность их связывания с помощью механизма «Ведущий – ведомый».

Важно. Настройка логики «Ведущий – ведомый» доступна только для свойств типа Ссылка .

Ведущее свойство определяет доступность значений ведомого. Например, свойство Регион является ведомым для свойства Страна . В результате при заполнении свойства Регион можно выбрать только регионы страны, указанной в свойстве Страна :

Чтобы настроить логику «Ведущий – ведомый» между свойствами:

- 1. В редакторе типа сущности, например типа справочника Город , откройте вкладку «Свойства».
- 2. Перейдите к свойству, список доступных значений которого нужно фильтровать, например Регион . Оно будет ведомым:
- 3. Настройте логику, заполнив поля:

• Ведущее свойство , по которому фильтруется список доступных значений. Оно будет ведущим. Например, для свойства Регион ведущим может являться свойство Страна , и при заполнении поля Регион для выбора доступны только регионы страны, указанной в соответствующем поле.

Свойство может иметь одно ведущее свойство и несколько ведомых. При изменении ведущего его связь с ведомыми сбрасывается.

• Связано с . Свойство типа сущности, указанного в поле *Тип сущности , которое идентично свойству в поле Ведущее свойство . Устанавливает связь между ведущим и ведомым свойствами.

- ПРИМЕЧАНИЕ. Значения полей Ведущее свойство и Связано с очищается при изменении поля *Тип свойства .

В результате свойства связываются, и для фильтрации списка доступных значений ведомого свойства используется свойство, указанное в поле Связано с . Например, при выборе региона в карточке типа сущности Город можно выбрать только регионы страны, указанной в поле Страна .

Также, если настроена логика «Ведущий – ведомое», при заполнении ведомого свойства, ведущее автоматически подставляется.

С помощью логики можно выстроить цепочку свойств, например Страна – Регион – Город . При этом каждое ведомое свойство фильтруется только по его непосредственному ведущему. В таком случае, если заполнить свойство Страна , а Регион оставить пустым, список доступных значений свойства Город не фильтруется.
