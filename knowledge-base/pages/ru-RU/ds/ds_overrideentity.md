# Шаг 1. Перекрытие типа документа

- 1. Создайте решение, например Purchases .
- 2. В дереве решения рядом с узлом «Перекрытые элементы» нажмите на , затем выберите пункт Документ .
- 3. В открывшемся окне в выпадающем списке выберите тип сущности базового решения Sungero.Contracts.Contract :
- 4. Нажмите на кнопку Перекрыть . В результате договор появляется в дереве решения в пункте Перекрытые элементы , и открывается его редактор.
- 5. Перейдите в редактор созданного решения и добавьте зависимость от модуля «Договоры» ( Sungero.Contracts ).
- ВАЖНО. Если зависимости не указаны, при публикации могут возникнуть ошибки.
- 6. В редакторе перекрытого типа документа на вкладке «Свойства» добавьте свойство Резидент (Resident) с типом Логическое :
- СОВЕТ. Для совместимости со стандартной прикладной разработкой рекомендуется для свойств задавать уникальные имена.
- 7. Чтобы при создании договора флажок Резидент по умолчанию был снят, перейдите на вкладку «Главная» и в группе «События» рядом со ссылкой Создание нажмите на кнопку . В открывшемся редакторе вставьте код:

```csharp
public override void Created(Sungero.Domain.CreatedEventArgs e)
{
base.Created(e);
_obj.Resident = false;
}
```

- 8. Опубликуйте разработку.
- 9. В веб-клиенте настройте новое представление формы карточки договора и добавьте рядом с полем Контрагент флажок Резидент .
- Примечание. Чтобы для договора применялось новое представление, для него нужно указать приоритет выше, чем у представления по умолчанию.
