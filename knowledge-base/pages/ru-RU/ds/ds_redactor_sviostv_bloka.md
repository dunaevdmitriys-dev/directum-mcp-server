# Редактор свойств блока

Для уточнения логики процесса, указания исполнителей, сроков и другой информации о блоке используются свойства. Существуют стандартные свойства, например Автор (Author) или Описание (Description) для блоков типов «Задание» и «Уведомление». Такие свойства можно отображать или скрывать в настройках. В дополнение к стандартным можно создавать новые свойства .

Чтобы передавать данные из событий в параметры процесса, в блоках можно использовать выходные свойства .

Примечание. К свойствам можно обращаться в программном коде.

Для настройки перехода по разным веткам схемы добавьте дополнительные результаты выполнения для блоков типов «Скрипт» и «Мониторинг».

## Создание свойства

Добавьте свойства блока. Для этого: 1. В настройках блока перейдите к его свойствам и нажмите на кнопку : 2. В открывшемся редакторе заполните именование и параметры свойства: *Имя . Должно быть уникальным в рамках одного модуля, может состоять из букв латинского алфавита и цифр, начинаться с буквы. *Отображаемое имя . Имя свойства, которое отображается в проводнике. Для локализации имени наведите на поле и нажмите кнопку . *Тип свойства . Определяет, какие данные могут быть значением свойства. В выпадающем списке выберите один из типов свойств : строка, целое, логическое, вещественное, дата, ссылка, коллекция ссылок, перечисление. *Тип сущности . Сущности указанного типа являются значениями для указанных параметров. Поле доступно для параметров типов Ссылка или Коллекция ссылок . Значение перечисления . Список возможных значений перечисления. Доступно только для типа Перечисление . Отображать в настройках . Свойство будет доступно для настройки схемы процесса в проводнике веб-клиента. ПРИМЕЧАНИЕ. Если свойство устарело, то снимите флажок. Блоки, в которых задано свойство, продолжат его учитывать. При этом аналитик не сможет изменить свойство в существующем блоке и задать его в новом. Обязательное . Установите флажок, если свойство нужно сделать обязательным для заполнения. Добавьте обработчики событий . Код обработчиков событий выполняется, когда вызывается соответствующее событие. В открывшемся редакторе напишите код обработчика события.

## Создание выходного свойства

Иногда при настройке схемы процесса необходимо передать данные из событий блока в параметры процесса. Для этого используются выходные свойства блока. Например, в них можно записывать результат преобразования приказа в формат PDF: ссылку на документ или текст ошибки. К выходным свойствам можно обращаться через вычисляемое выражение при настройке схемы процесса и в программном коде. Для этого в параметрах процесса при завершении блока, в вычисляемом выражении укажите элемент Блок и выберите нужное свойство: Чтобы создать выходное свойство: 1. В блоке перейдите к выходным свойствам и нажмите на кнопку : 2. В открывшемся редакторе заполните именование и параметры выходного свойства: *Имя . Должно быть уникальным в рамках одного модуля, может состоять из букв латинского алфавита и цифр, начинаться с буквы. *Отображаемое имя . Имя выходного свойства, которое отображается в проводнике. Для локализации имени наведите на поле и нажмите кнопку . *Тип свойства . Определяет, какие данные могут быть значением свойства. В выпадающем списке выберите один из типов свойств : строка, целое, логическое, вещественное, дата, ссылка, коллекция ссылок, перечисление. *Тип сущности . Поле доступно только для параметра типа Ссылка или Коллекция ссылок . Сущности указанного типа являются значениями для указанных параметров.

Результаты выполнения в блоках «Скрипт» и «Мониторинг»

```csharp
public virtual void ScriptBlockExecute()
{
var document = _block.Document;
try
{
var conversionResult = Sungero.Docflow.PublicFunctions.Module.ConvertToPdf(document);
if (conversionResult.HasErrors)
{
Logger.Debug($"Document (ID={document.Id}, version={document.LastVersion.Id}. Converting to pdf failed, message: {conversionResult.ErrorMessage}");
_block.OutProperties.ExecutionResult = ExecutionResult.ConvertError;
}
else
{
_block.OutProperties.ExecutionResult = ExecutionResult.Success;
}
}
catch (Exception ex)
{
Logger.Debug($"Exception occurred when converting to pdf, message: {ex.Message}");
_block.OutProperties.ExecutionResult = ExecutionResult.ConvertError;
}
}
```
