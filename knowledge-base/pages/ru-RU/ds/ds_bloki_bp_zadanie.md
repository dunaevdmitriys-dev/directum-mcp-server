# Блок «Задание»

Блок типа «Задание» (AssignmentBlock) предназначен для создания заданий. Выход из блока происходит при выполнении или прекращении всех заданий, созданных в рамках блока.

Свойства блока

Свойство | Описание
Заголовок ( Title ) | Заголовок блока, который отображается в схеме задачи
Описание (Description) | Поясняющий текст, который отображается на панели настроек блока на схеме варианта процесса. ПРИМЕЧАНИЕ. В прикладном коде свойство доступно только для чтения
Инициатор (Author) | Инициатор задания
Исполнители ( Performers ) | Список исполнителей. Для выбора доступны роли и наследники от групп. Совет. Sid (идентификатор субъекта прав) можно узнать в системной таблице Sungero_Core_Recipient
Тема ( Subject ) | Тема задания, которую видят пользователи в проводнике
Тема в переписке (ThreadSubject) | Тема задания в переписке
Текст ( Text ) | Текст задания, который видят пользователи в проводнике
Инструкция (Instruction) | Инструкция к заданию
Результат отсутствия исполнителей ( NoPerformersResult ) | Результат отсутствия исполнителей – это результат выполнения блока, если по нему не было создано ни одного задания. Возможные значения: • Не задан ; • результат выполнения для задания, которое указанно в параметре *Тип задания . Например, Выполнить , На доработку или Отложить ; • Результат: Нет исполнителей . Используйте, если нужно добавить переход из блока в виде отдельной стрелки в случае отсутствия исполнителей. На схеме появляется возможность указать значение Нет исполнителей для исходящей из блока стрелки; • Иначе
Параллельные задания ( IsParallel ) | Признак, того, что задания по блоку создаются одновременно или друг за другом. Возможные значения: • True – параллельно ; • False – последовательно . Значение по умолчанию
Конкурентные задания ( IsCompetitive ) | Признак, того, что в блоке создаются конкурентные задания. Возможные значения: • True – создаются ; • False – не создаются . Значение по умолчанию
Срок выполнения ( RelativeDeadline ) | Срок выполнения задания. При необходимости в программном коде с помощью свойств RelativeDeadlineDays и RelativeDeadlineHours в программном коде можно указать срок выполнения в рабочих днях или в часах
Учитывать отсутствия (IsWithAbsences) | Признак того, что срок выполнения сдвигается на период отсутствия сотрудников. Возможные значения: • True – отсутствия учитываются; • False – не учитываются. Значение по умолчанию . Если срок задается с помощью вычисляемого выражения, флажок скрывается. В этом случае для учета отсутствий необходимо использовать соответствующие функции
Срок прекращения ( RelativeStopAssignmentsDeadline ) | Срок принудительного прекращения задания. При необходимости с помощью свойств RelativeStopDeadlineDays и RelativeStopDeadlineHours в программном коде можно задавать срок прекращения в днях или в часах
Представление по умолчанию (DefaultViewForm) | Представление формы , которое отображается при открытии карточек отдельно от списка. Если свойство не задано, подбирается наиболее подходящее представление
Представление в списке (EmbeddedViewForm) | Представление формы для карточки в списке . Если свойство не задано, отображается представление по умолчанию

## Серверные события блока

Событие | Назначение | Аргументы события
Старт блока (<Имя блока>Start) | Заполнение свойств выполняемого блока. Событие вызывается при старте обработки блока | _obj – задача, в рамках которой обрабатывается блок; _block – блок, событие которого исполняется
Старт задания (<Имя блока>Start<Тип задания>) | • заполнение свойств, которые должны иметь разные значения у разных заданий. Например, разные сроки выполнения задания у разных исполнителей в составном поручении; • заполнение свойств задания, которые нельзя заполнить через свойства блока. Например, свойства типа Текст , Картинка , Бинарные данные . Событие вызывается до старта задания (после создания и заполнения свойств задания) и до сохранения в базу данных. Событие выполняется для каждого задания, созданного в рамках блока | _obj – задача, в рамках которой обрабатывается блок; assignment – задание, созданное в рамках блока; _block – блок, в рамках которого выполняется событие
Выполнение задания (<Имя блока>CompleteAssignment) | Актуализация свойств задачи или других заданий в зависимости от свойств выполненного задания. Событие вызывается при завершении каждого из заданий, созданного в рамках блока | _obj – задача, в рамках которой обрабатывается блок; assignment – выполненное задание; _block – блок, в рамках которого выполняется событие
Конец блока (<Имя блока>End) | Актуализация свойств задачи при завершении всех заданий, созданных по блоку. Событие вызывается, когда завершатся все задания в рамках блока. ПРИМЕЧАНИЕ . Изменение свойств объектов, заданное в веб-клиенте при завершении блока , происходит после выполнения события «Конец блока» | _obj – задача, в рамках которой обрабатывается блок; _block – блок, в рамках которого выполняется событие; createdAssignments – список заданий, созданных в рамках блока
Все серверные события блока | _block.ProcessStagesDisplayMode – режим отображения блока в этапах процесса. Возможные значения: • Show – пока не выполнен. Блок отображается, если он выполняется в текущий момент или будет выполняться в дальнейшем; • ShowInWork – в работе. Блок отображается, если он выполняется в текущий момент; • Hide – никогда. Блок скрыт. совет. Изменять режим отображения рекомендуется в веб-клиенте через свойство блока Отображать в этапах процесса

## Пример. Актуализация свойств задания в зависимости от результата его выполнения

```csharp
public virtual void MyAssignmentBlockCompleteAssignment(Sungero.Docflow.IMyAssignment assignment)
{
// Переадресовать задание, если оно выполнено с результатом «Переадресовать».
if (assignment.Result == Sungero.MyModule.MyAssignment.Result.Forward)
{
assignment.Forward(assignment.ForwardPerformer, ForwardingLocation.Next);
}
else if (assignment.Result == Sungero.MyModule.MyAssignment.Result.Approve)
{
// В список исполнителей добавить сотрудников, которые выполнили задание c
// результатом «Утверждено».
_obj.ApprovedPerformers.Add(assignment.Performer)
}
}
```

## Клиентские события блока

Событие | Назначение | Аргументы события
Обновление блока (<Имя блока>RefreshBlock) | Задание логики для отображения свойств блока в зависимости от условий. Вызывается при открытии схемы процесса в редакторе и изменении свойств блока | _block – блок, в рамках которого выполняется событие _block.<название свойства>.Value – значение свойства; _block.<название свойства>.IsEmpty() – проверка, заполнено ли свойство; _block.<название свойства>.IsExpression() – проверка, является ли свойство вычисляемым выражением; _block.State.Properties.<название свойства>.<состояние> – состояние свойства. Возможные значения: • isVisible – видимое; • isEnabled – доступное; • isRequired – обязательное _block.ProcessStagesDisplayMode – режим отображения блока в этапах процесса. Возможные значения: • Show – пока не выполнен. Блок отображается, если он выполняется в текущий момент или будет выполняться в дальнейшем; • ShowInWork – в работе. Блок отображается, если он выполняется в текущий момент; • Hide – никогда. Блок скрыт. Совет. Изменять режим отображения рекомендуется в веб-клиенте через свойство блока Отображать в этапах процесса

## Пример. Отображение выпадающего списка в зависимости от того, установлен ли флажок

```csharp
public virtual void AssignmentBlockRefresh()
{
// Если флажок PropertyBool установлен, то отобразить выпадающий список PropertyEnum
_block.State.Properties.PropertyEnum.IsVisible = _block.PropertyBool.Value.GetValueOrDefault();
}
```
