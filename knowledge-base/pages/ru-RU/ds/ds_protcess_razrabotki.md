# Процесс разработки

Directum RX поставляется с базовым решением Sungero.Directum RX – стандартной прикладной частью, с помощью которой автоматизируется работа с официальными документами классического делопроизводства, финансово-хозяйственными документами и договорами, документами для совещаний и проектов, обмен электронными юридически значимыми документами с контрагентами и многое другое.

При необходимости базовое решение легко настраивается под особенности организации без программирования с использованием подхода no-code. Однако, если требуется глубокая адаптация под бизнес-задачи, например, из-за специфики отрасли, тогда можно модифицировать базовое решение под себя с помощью среды разработки Development Studio .

Порядок модификации элементов разработки и публикации их в продуктивную систему представлен на схеме:

- 1. Запустите среду разработки .
- 2. Создайте решение и в нем:

• измените стандартные возможности Directum RX с помощью механизма перекрытия ;

• разработайте свои модули и типы сущностей с помощью механизма наследования .

- Также вы можете добавить сторонние компоненты, чтобы с их помощью адаптировать интерфейс веб-клиента под потребности компании.
- ПРИМЕЧАНИЕ . Если используются мобильные приложения Directum Solo или Directum Jazz учитывайте особенности разработки для мобильных решений . Это необходимо, чтобы реализовать поддержку собственной разработки в мобильных приложениях.
- 3. Соберите решения , чтобы проверить, что код написан верно и в нем нет несовместимостей.
- 4. Опубликуйте решения в локальную систему для проверки работы изменений. При этом вы можете использовать разные конфигурации: упрощенную, чтобы быстро проверить разработанное решение, или полную для просмотра того, как решение работает в условиях, близких к реальным. Подробнее см. раздел «Конфигурирование настроек» ( Windows , Linux ).

Примечание . Для обеспечения стабильной работы Directum RX проверка и отладка изменений выполняется в отдельной локальной системе , в то время как пользователи работают в продуктивной системе .

- 5. Создайте настройки по умолчанию для бизнес-процессов и интерфейса.
- 6. При необходимости выполните отладку , исправьте разработку и заново опубликуйте.
- 7. Создайте пакет разработки с исполняемыми файлами для обновления продуктивной системы.

Важно . Перед публикацией создайте резервную копию БД. Это позволит восстановить данные при возникновении ошибок.

8. Опубликуйте пакет разработки в продуктивный экземпляр Directum RX с помощью инструмента Directum Launcher.

9. Для работы с обновленным системой необходимо перезапустить клиентское приложение. Это происходит автоматически, если приложение закрыто.

С помощью среды разработки вы также можете настроить интеграцию Directum RX с другими информационными системами вашей компании и таким образом создать единую корпоративную экосистему. Для обращения к внешним приложениям на компьютере или к локальным файлам в операционной системе можно использовать утилиту RxCmd .
