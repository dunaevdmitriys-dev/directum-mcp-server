# Запрещенные возможности

Запрещается:

- использовать анонимные типы
- генерировать, перехватывать и гасить исключения
- приводить к типу с помощью оператора as
- проверять тип с помощью оператора is
- создавать объекты с помощью оператора new
- использовать статические поля в прикладном коде
- использовать модификатор параметра out

Анонимные типы

Запрещается использовать анонимные типы: • var obj = new { Name = 'Name1' } • Select ( p = > new { Name = p . Name }) Ограничение связано с тем, что они не передаются между сервером и клиентским приложением. .Вместо анонимных типов используйте структуры .

Генерация, перехват и гашение исключений

Запрещается генерировать родные исключения .NET. Также запрещается перехватывать и гасить исключения класса Exception: catch ( Exception ex ) { } Ограничения связаны с тем, что может быть нарушена логика работы ядра Directum RX. При этом разрешается генерировать и перехватывать прикладные и предметные исключения: try { throw AppliedCodeException . Create ( Resources . Message , Plans . Resource . Description ); } catch ( AppliedCodeException ex ) { } Важно. Перехватывайте предметные исключения, если вы точно знаете , что это за исключение . Для обработки и передачи исключений используйте конструкцию: catch ( Exception ex ) { throw ex ; }

Приведение к типу с помощью оператора as

Запрещается приведение к типу с помощью оператора as: var employee = entity as IEmployee Ограничение связано с использованием ORM-библиотеки NHibernate . Поэтому возможна ситуация, когда сущность приводится к интерфейсу, который она не реализует. В этом случае код не работает. Вместо этого для безопасного приведения к типу используйте метод Employees.As(entity) .

Проверка типа с помощью оператора is

Запрещается проверка типа с помощью оператора is: if ( entity is IEmployee ) Ограничение связано с использованием ORM-библиотеки NHibernate . Поэтому возможна ситуация, когда сущность приводится к интерфейсу, который она не реализует. В этом случае код не работает. Вместо этого для безопасной проверки используйте метод Employees.Is(entity) .

Создание объектов с помощью new

Запрещается создавать объекты с помощью оператора new : • var result = new Tuple < long , DateTime >( id , date ) • var ex = new ArgumentOutOfRangeException ( "date" ) Ограничение связано с тем, что через оператор new разрешается создавать только коллекции. Вместо этого используйте метод создания для класса, в котором нужно создать объект, например Create() .

Использование статических полей

Запрещается использовать статические поля в прикладном коде: static List < double ?> list = new List < double ?>(); Ограничение связано с тем, что поля не поддерживаются на уровне платформы. Вместо них используйте свойства .

Модификатор параметра out

Запрещается использовать модификатор параметра out : public bool TryGetEmployeeForTask ( out Sungero . Company . IEmployee employee ) Ограничение связано с тем, что модификатор не поддерживается в среде разработки. Вместо него используйте структуры .
