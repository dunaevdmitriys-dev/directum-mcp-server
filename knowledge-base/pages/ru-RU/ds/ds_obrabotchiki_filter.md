# Обработчики событий

События свойств сущности
Фильтрация | Filtering
Получение данных | DataQuery
Предварительная фильтрация | PreFiltering
Проверка фильтра | ValidateFilterPanel

Кроме того, в коде событий можно использовать аргумент _filter . Он представляет собой объект с параметрами фильтрации и хранит значения, выбранные пользователем в контролах. С помощью значений можно фильтровать записи по заданным критериям.

## Получение данных

Обработчик события для фильтрации содержимого папки. Доступен, если в редакторе папки на вкладке «Основное» заполнено поле Основное содержимое . В коде обработчика задаются:

- состав содержимого папки. Например, в ней могут быть только задания на подписание или документы, отправленные контрагентам;
- условия фильтрации, если настроена панель фильтрации.

## Проверка фильтра

Обработчик события для валидации панели фильтрации. Например, с помощью него можно проверять параметры фильтрации, которые пользователь вводит.

СОВЕТ. Используйте обработчик, чтобы предотвратить отправку запросов, которые возвращают тысячи записей и замедляют работу системы.

Обработчики событий Проверка фильтра использует аргумент _filter для обращения к панели фильтрации и ее элементам. Подробнее см. пример настройки фильтрации записей в папке.

## Аргумент _filter

Текущее состояние панели фильтрации хранится в аргументе _filter . В обработчиках событий аргумент используется для обращения к панели фильтрации и ее элементам.

Аргумент _filter отвечает за настройку фильтрации записей по заданным критериям. Им соответствуют свойства, которые связаны с добавленными на панель фильтрации контролами. Поэтому, например, при фильтрации списка входящих документов список доступных свойств один, при фильтрации договоров – другой.

Критерии фильтрации для списка «Входящие документы»

Свойства

Имя | Тип | Описание
Counterparty | ICounterparty | Контрагент
DateRangeFrom | Nullable<DateTime> | Период дат с…
DateRangeTo | Nullable<DateTime> | Период дат по…
Department | IDepartment | Подразделение
DocumentKind | IDocumentKind | Вид документа
DocumentRegister | IDocumentRegister | Журнал регистрации
Last90Days | bool | Последние 90 дней
LastMonth | bool | Последние 30 дней
LastWeek | bool | Последние 7 дней
ManualPeriod | bool | Произвольный период, введенный вручную
NotRegistred | bool | Статус документа «Зарегистрирован»
Registred | bool | Статус документа «Не зарегистрирован»

Чтобы проверить, включена ли панель фильтрации, в начало обработчика события добавляется проверка вида:

```csharp
if (_filter == null)
return query;
```

Пример. Фильтрация списка входящих документов по критериям

```csharp
public override IQueryable<T> Filtering(IQueryable<T> query, Sungero.Domain.FilteringEventArgs e)
{
// Проверить, что панель фильтрации включена.
if (_filter == null)
return query;
// В контроле "Ссылка на сущность" отфильтровать по журналу регистрации.
// Выбрать значение из справочника "Журналы регистрации".
if (_filter.DocumentRegister != null)
query = query.Where(d => d.DocumentRegister == _filter.DocumentRegister);
// В контроле "Набор флажков" отфильтровать по статусу документа «Зарегистрирован».
if (_filter.Registered)
query = query
.Where(d => (_filter.Registered && d.RegistrationState == OfficialDocument.RegistrationState.Registered);
return query;
}
```
