# Бизнес-процессы

## Review – рассмотрение документов

Процессы рассмотрения входящих, исходящих или внутренних документов, по которым выносится резолюция или утверждается проект резолюции, подготовленный помощником.

В мобильном приложении:

- если в задание вложен проект поручения, отображается вкладка «Поручение» с поручением, автором которого является текущий работник или замещаемый сотрудник;
- если проект поручения не вложен, открывается вкладка «Вложения» на основном документе по проекту.

Пользователь может самостоятельно настроить вкладку, на которой открываются задания с вынесением резолюции. В Directum Solo пользователь может настроить отображение проектов поручений в задании: отображать только свои или все созданные поручения. Подробнее см. в инструкциях по установке и настройке мобильных решений, раздел «Настройка».

## Control – контроль за исполнением поручений

В мобильном приложении вкладка с поручением не отображается. Задание открывается на переписке.

## Execution – исполнение поручений

В мобильном приложении по умолчанию задание открывается на вкладке с резолюцией, если иное не задано в настройках приложения.

## Regulations – работа с документами, недоступными для редактирования

К документам, которые не должны быть доступны для редактирования, относятся, например, нормативно-правовые акты.

Если такой документ вложен в задание, мобильное приложение запрещает его изменение. Кнопка Редактировать документ скрывается. Следует учитывать, что документ может быть изменен этим же пользователем в рамках другой задачи .

## Approval – согласование документов

Мобильное приложение при открытии задания открывает вкладку просмотра документа.

## Thread – продление срока по поручению

Мобильное приложение при открытии задания открывает вкладку с перепиской. Отображается вкладка «Поручение» с поручением, по которому продлевается срок.
