# Сторонние компоненты

Чтобы адаптировать интерфейс системы под потребности компании, с помощью сторонних компонентов в решения можно добавлять свои контролы . Для этого:

- 1. Реализуйте компонент с необходимыми контролами. Для этого скачайте пример исходного кода стороннего компонента из репозитория GitHub и доработайте его.
- ПРИМЕЧАНИЕ. Чтобы настройки веб-сервера можно было передавать в сторонний компонент, используйте параметр REMOTE_COMPONENT_SETTINGS . Подробнее см. в раздел «Веб-сервер» ( Linux , Windows ).
- 2. Соберите сторонний компонент. Для этого предварительно установите среду выполнения JavaScript Node.js и менеджер пакетов npm. Затем перейдите в папку с исходным кодом стороннего компонента и с помощью командной строки выполните:

npm run build:release

- Подробный список команд для работы с файлами стороннего компонента см. в репозитории GitHub, раздел README .
- 3. Запустите среду разработки.
- СОВЕТ. Чтобы информация о компонентах обновлялась сразу, вносите все изменения при запущенной Development Studio. Если она закрыта, информация о компонентах обновится при следующем запуске среды разработки.
- 4. В репозитории среды разработки перейдите в папку сторонних компонентов решения <Имя решения>.Components. Например, SolutionPurchases\SolutionPurchases.Components.
- 5. Добавьте папку с файлами стороннего компонента. Ее имя должно состоять из букв латинского алфавита и цифр и начинаться с буквы.
- ВАЖНО. В папке стороннего компонента должен быть файл metadata.json, он формируется при сборке компонента. Также значение свойства publicName , которое указывается в файле, должно быть уникальным в рамках решения.
- 6. Опубликуйте решение .

В результате сторонние контролы можно добавлять в проводнике в редакторах форм и обложек .

Чтобы удалить контролы:

- 1. В репозитории перейдите в папку сторонних компонентов решения <Имя решения>.Components.
- 2. Удалите папку стороннего компонента контролов.

Чтобы изменить контролы:

- 1. В репозитории перейдите в папку сторонних компонентов решения <Имя решения>.Components.
- 2. Внесите необходимые изменения в папку стороннего компонента контролов.
- ВАЖНО. Если необходимо только изменить сторонние контролы, не удаляйте папку их компонента. Ее удаление и создание папки с тем же именем расценивается как добавление нового компонента с другими идентификаторами.

## Версионирование контрола

В веб-клиенте можно одновременно работать с несколькими версиями одного контрола. Например, чтобы критичные доработки стороннего компонента не повлияли на перекрытые решения, рекомендуется вносить изменения в новую версию.

При использовании примера стороннего компонента из репозитория GitHub , чтобы создать новую версию контрола:

- 1. В файле стороннего компонента component.manifest.js в параметре componentVersion измените номер версии.
- 2. Соберите проект стороннего компонента.
- 3. Добавьте папку нового компонента в папку компонентов решения.
- СОВЕТ. Чтобы отличать версии контрола, указывайте в названиях их номера.
- 4. Опубликуйте решение .

В результате в редакторе форм и обложек можно использовать и созданную ранее версию контрола, и новую.
