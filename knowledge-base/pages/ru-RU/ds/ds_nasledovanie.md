# Наследование

Механизм наследования позволяет на рабочем слое разработки в модуле решения создавать новые типы сущностей, которые наследуют структуру данных, свойства, действия и логику работы от других типов сущностей. Использование механизма сокращает время на разработку и уменьшает дублирование кода между схожими типами сущностей.

Тип сущности, от которого наследуются, называется абстрактным. При создании типа сущности для него всегда указывается абстрактный тип. Например, при создании договора в качестве абстрактного выбирается документ. Договор наследует от него версии и возможность их подписания. При этом в договор добавляется свое свойство Контрагент . Подробнее про наследование договорных документов см. в разделе «Схема наследования договорных документов» .

Абстрактный тип определяет поведение и возможности наследника, при этом можно расширить и изменить унаследованное. Подробнее см. раздел «Что можно изменять в наследнике» .

В системе есть абстрактные типы сущностей:

- платформы . Например, в среде разработки есть стандартные абстрактные типы «Задача» ( Sungero.Workflow.Task ), «Задание» ( Sungero.Workflow.Assignment ), «Уведомление» ( Sungero.Workflow.Notice ), «Задание на приемку» ( Sungero.Workflow.ReviewAssignment ), «Электронный документ» ( Sungero.Content.ElectronicDocument ), «Справочник» ( Sungero.CoreEntities.DatabookEntry ) и т.д. На их основе можно создать свои типы сущностей;
- базовых решений . Например, в стандартном решении Directum RX есть абстрактный тип документа «Базовый договор» ( ContractBase ). На его основе можно создать типы документов «Договор доходный», «Договор расходный» и т.д.
- При наследовании от прикладных абстрактных типов стоит учитывать, как с ними будут работать отчеты, задачи, списки и вычисляемые папки базовых решений. Например, наследники, созданные на основе базового типа:

• «Входящие документы» ( IncomingDocumentBase ) попадают в список «Входящие документы» модуля «Делопроизводство» и в отчет «Журнал входящих документов»;

• «Исходящие документы» ( OutgoingDocumentBase ) попадают в список «Исходящие документы» модуля «Делопроизводство»;

• «Внутренние документы» ( InternalDocumentBase ) попадают в список «Внутренние документы» модуля «Делопроизводство» и в отчет «Журнал внутренних документов»;

• «Бухгалтерские документы» ( AccountingDocumentBase ) попадают в список «Финансовые документы» модуля «Финансовый архив», а также доступны при выгрузке документов из финансового архива;

• «Договоры» ( ContractBase ) попадают в папку «Реестр договоров» модуля «Договоры», учитываются в рассылке о завершении срока действия договоров, а также доступны при выгрузке договоров из финансового архива.

• ваших решений . Например, в решении создан абстрактный тип документа «Входящее письмо» (MyInternalDocumentBase). На его основе можно создать типы документов «Входящее письмо контрагента», «Входящие заявки» и т.д.

Чтобы тип сущности стал абстрактным, при его создании необходимо установить флажок Абстрактный тип . При этом создавать можно только абстрактные типы справочников и документов.

- Важно . В системе можно работать только конкретными типами сущностей . Например, в вашем решении есть абстрактный тип документа «Исходящая заявка», от которого создан наследник не абстрактного типа «Исходящая заявка на закупку». В таком случае в проводнике пользователь может создавать документы только типа «Исходящая заявка на закупку».
