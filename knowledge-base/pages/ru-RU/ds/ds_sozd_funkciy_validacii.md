# Создание функций валидации для правил ввода

Функции используются для проверки значения, которые пользователь ввел в поле диалога. Созданные функции отображаются в правиле ввода в выпадающем списке Функции валидации .

При создании функций учитывайте их особенности .

## Особенности создания функций

Функции валидации для правил ввода:

- создаются в клиентском коде с использованием атрибута LocalizeFunction ;
- должны принимать единственный аргумент с типом InputRuleValidationEventArgs ;
- должны возвращать значение с типом void .

## Порядок создания функции

Предположим, необходимо написать функцию, которая проверяет ИНН по контрольной сумме. В случае ошибки пользователю отображается текстовое сообщение. Для этого:

- 1. Создайте клиентскую функцию модуля. В этом случае она не будет привязана к типу сущности.
- Совет. Если функцию нужно привязать к определенному типу сущности, создайте ее в редакторе типа сущности.

2. В редакторе кода укажите нужное имя функции, добавьте комментарий и напишите код для проверки введенного значения. Перед объявлением функции укажите атрибут LocalizeFunction :

```csharp
[LocalizeFunction("Проверка ИНН", "Проверяет корректность ИНН (10 или 12 цифр) по контрольной сумме")]
public void ValidateINN(Sungero.Domain.Client.InputRuleValidationEventArgs e)
{
//Валидации ИНН организации(10 цифр)
var inn = e.Value.Trim();
if(inn.Length == 10)
{
int[] coefficients10 = { 2, 4, 10, 3, 5, 9, 4, 6, 8 };
int checksum10 = 0;

for (int i = 0; i < 9; i++)
checksum10 += coefficients10[i] * int.Parse(inn[i].ToString());
int controlNumber = checksum10 % 11;
if (controlNumber > 9)
controlNumber %= 10;

if (controlNumber != int.Parse(inn[9].ToString()))
e.AddError("Некорректный ИНН. Проверьте правильность ввода");
}

//Валидации ИНН ИП и ФЛ (12 цифр)
if(inn.Length == 12)
{
int[] coefficients11 = { 7, 2, 4, 10, 3, 5, 9, 4, 6, 8 };
int[] coefficients12 = { 3, 7, 2, 4, 10, 3, 5, 9, 4, 6, 8 };

int checksum11 = 0;
for (int i = 0; i < 10; i++)
checksum11 += coefficients11[i] * int.Parse(inn[i].ToString());
int controlNumber11 = checksum11 % 11;
if (controlNumber11 > 9)
controlNumber11 %= 10;

int checksum12 = 0;
for (int i = 0; i < 11; i++)
checksum12 += coefficients12[i] * int.Parse(inn[i].ToString());
int controlNumber12 = checksum12 % 11;
if (controlNumber12 > 9)
controlNumber12 %= 10;

if (controlNumber11 != int.Parse(inn[10].ToString()) || controlNumber12 != int.Parse(inn[11].ToString()))
e.AddError("Некорректный ИНН. Проверьте правильность ввода");
}
}
```

- Подробнее об алгоритме проверки ИНН см. на справочном портале Госуслуг статью «Валидация» .
- 3. Сохраните изменения.
- 4. Опубликуйте разработку .
- 5. Добавьте созданную функцию в правило ввода . Для дополнительной проверки вводимого значения используйте:

• маску 000000000000, чтобы пользователь мог ввести максимум 12 символов;

• регулярное выражение ^(?!(\d)\1{9,11}$)\d{10,12}$, чтобы проверить, что введенное значение состоит из 10 или 12 цифр.
