## Виджеты

### Типы виджетов

| Тип | Описание |
|-----|----------|
| Виджет с действиями | Список действий с значениями-счётчиками |
| Виджет с диаграммой | Кольцевая, круговая, гистограмма, линейный график |

### Создание виджета с действиями

1. Редактор модуля → узел «Виджеты» → Добавить
2. Свойства виджета:

| Свойство | Описание |
|----------|----------|
| `*Имя` | Уникальное имя (латиница + цифры) |
| `*Отображаемое имя` | Заголовок в проводнике |
| `Описание` | Описание при добавлении |
| `*Количество колонок` | Ширина (1–100) |
| `*Цвет` | Цвет по умолчанию |
| `Отображать виджет` | Видимость |

3. Добавить действия → Добавить → Действие:

| Свойство действия | Описание |
|-------------------|----------|
| `*Имя` | Уникальное в виджете |
| `Отображаемое имя` | Текст действия (локализуемый) |
| `*Обновление` | Период: при каждом обращении / в начале дня/недели/месяца/квартала/года |
| `*Размер шрифта` | Мелкий / Средний / Крупный |
| `*Цвет шрифта` | Серый или красный |
| `*Тип действия` | Показ списка сущностей / Разрабатываемое |

### Типы действий

**Показ списка сущностей:**
- Указывается `*Тип сущности`
- Опция «Значение показывает количество сущностей» — автоподсчёт
- Обработчик «Фильтрация» — фильтрует список

**Разрабатываемое:**
- Обработчики: «Получение значения» + «Выполнение»
- Полная свобода логики

### Вложенные действия и разделители

- Вложенные действия визуально отступают под родительским
- Разделители: тип «Отступ» или «Линия»

### Обработчики событий виджетов

**Фильтрация** — фильтрует список сущностей:

```csharp
public virtual IQueryable<Sungero.Content.IElectronicDocument>
    DocumentsTodayFiltering(
        IQueryable<Sungero.Content.IElectronicDocument> query)
{
  if (_parameters.ShowDocuments == Widgets.Documents.ShowDocuments.Read)
    query = query.WhereDocumentHistory(h =>
        h.Action == Sungero.CoreEntities.History.Action.Read);

  if (!_parameters.AllowDocumentWithoutBody)
    query = query.Where(h => h.HasVersions);

  return query.WhereDocumentHistory(h =>
      Equals(Users.Current, h.User) &&
      h.HistoryDate >= Calendar.Today);
}
```

**Получение значения** — вычисляет отображаемое число:

```csharp
public virtual string GetDocumentsPerDayValue()
{
  var documents = Sungero.Content.ElectronicDocuments.GetAll();

  if (_parameters.Show == Widgets.Documents.Show.Update)
    documents = documents.WhereDocumentHistory(h =>
        h.Action == Sungero.CoreEntities.History.Action.Update);

  return CalculateAverageDocumentsPerDay(documents);
}
```

**Выполнение** — логика при клике:

```csharp
public virtual void ExecuteDocumentsStatisticAction()
{
  var report = Docflow.Reports.GetDocumentStatisticReport();
  report.Open();
}
```

## Параметры виджетов

Позволяют пользователю выбирать отображаемые данные.

### Встроенные параметры

Создаются автоматически для каждого виджета:
- `Заголовок` (Title)
- `Цвет` (Color)

### Дополнительные типы параметров

| Тип | Описание |
|-----|----------|
| Логический | True / False |
| Перечисление | Выбор из списка значений |
| Ссылка | Ссылка на объект системы (сотрудник, подразделение). Множественный выбор |
| Период | Относительный период: Сегодня, Вчера, Последние 3/7/30 дней и т.д. |

Доступ в обработчиках: `_parameters.<ParameterName>`

## Виджеты с диаграммами

### Доступные типы диаграмм

- Кольцевая диаграмма
- Круговая диаграмма
- Гистограмма
- Линейный график

### Создание

1–3 шаги аналогичны виджету с действиями
4. Добавить → Диаграмма → задать:
   - `*Тип диаграммы`
   - `*Высота диаграммы` (1–100, относительно ширины)

### Обработчики

| Тип действия | Обработчики |
|-------------|-------------|
| Показ списка сущностей | Фильтрация + Получение значения |
| Разрабатываемое | Получение значения + Выполнение |
| Линейный график | Только Получение значения |

## Наборы виджетов

Набор — именованная коллекция виджетов для проводника. Доступ можно ограничить ролями.

### Создание

1. Редактор решения → узел «Наборы виджетов» → Добавить
2. Задать имя и отображаемое имя
3. Добавить виджеты:
   - **Разработанный виджет** — из модулей решения
   - **Виджет типа сущности** — привязка к списку сущностей (например, Сотрудники)
   - **Виджет папки** — привязка к вычисляемой папке (например, Избранное)
4. Расположить виджеты drag-and-drop в области предпросмотра
5. Изменить значения параметров по умолчанию через кнопку «Параметры»

### Ограничение доступности

```csharp
// Обработчик «Ограничение доступности» набора виджетов.
private static bool CanApplyTopManager()
{
  return IncludedInRole("Руководители подразделений");
}

public static bool IncludedInRole(string roleName)
{
  var role = Roles.GetAll(r => r.Name == roleName).FirstOrDefault();
  if (role == null)
    return false;
  return Users.Current.IncludedIn(role);
}
```

**Без обработчика** — набор виджетов доступен всем пользователям.

## Обложки (Covers)

### Концепция

Обложка — стартовая страница модуля в проводнике. Содержит общую информацию и часто используемые действия. Разным ролям можно показывать разные обложки.

### Настройка

**Рекомендуется** настраивать обложки в **веб-клиенте**, а не в среде разработки (поддерживается для обратной совместимости).

### Перекрытие обложки модуля в DDS

| Правило | Описание |
|---------|----------|
| Не удаляйте стандартные группы действий | Некоторые функции доступны только с обложки |
| Скрытая группа подавляет новые действия | Обновления добавляющие действия в скрытую группу — не будут видны |
| Слияние при импорте | При импорте нескольких базовых решений группы и действия объединяются |

### Структура в метаданных (Module.mtd)

```json
"Cover": {
  "NameGuid": "d5201bd3-fdff-4015-a574-fd025fe78f1a",
  "Actions": [],
  "Groups": [],
  "Header": { "BackgroundPosition": "Stretch" },
  "Footer": { "BackgroundPosition": "Stretch" },
  "RemoteControls": []
}
```

### Настройка в settings/ (пакет разработки)

Файл `settings/<Module>/ModuleView/<guid>.json` содержит:
- Настройки обложки (Cover)
- Список получателей (Recipients) — кто видит обложку
- Контекст клиента (DefaultContext / External client)
- Статус (Draft / Active)

## Мини-приложения Directum Omni

Для создания мини-приложения в суперприложении Omni:

1. Создать представление модуля с контекстом «Внешний клиент»
2. Настроить обложку модуля — она показывается при выборе мини-приложения
3. В заголовке обложки задать иконку — она станет иконкой мини-приложения
4. Настроить формы и списки для внешнего контекста
5. Переключить статус представления на «Активный»

**Ограничения Omni:**
- Действия формирования отчётов не отображаются
- Табличные части показывают «Не поддерживается»
- Сортировка и фильтрация не поддерживаются

## Навигация: структура проводника

### ExplorerTreeOrder в Module.mtd

Порядок элементов в дереве проводника задаётся в метаданных:

```json
"ExplorerTreeOrder": [
  {
    "ItemId": "b6eb442d-0380-47d1-b70f-683840ddc4d3",
    "PreviousItemId": "256907cc-abdc-47a6-b9c3-a13fc2ed11b6"
  }
]
```

### ExplorerModule (settings)

Файл `settings/<Module>/ExplorerModule/<guid>.json` — настройки навигационного модуля:
- Привязка к модулю
- Локализация имени модуля в проводнике
