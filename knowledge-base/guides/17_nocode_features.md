# NoCode-настройка системы

> Источник: `nc_nocode.htm`, `nc_bazovye_bloki.htm`, `nc_varianty_protsessov.htm`, `nc_vychislyayemyye_roli.htm`, `nc_nastroyka_spiskov.htm`, `nc_nastroyka_dialogov.htm`, `nc_nastroyka_kartochki_i_deystviy.htm`, `nc_prosmotr_sostoyania_blokov.htm`

## Обзор

No-code подход позволяет настраивать бизнес-процессы и интерфейс системы визуально, без написания кода. Изменения применяются «на лету».

### Пять направлений NoCode-настройки

| Область | Что настраивается |
|---------|------------------|
| Бизнес-процессы | Схемы процессов (блоки, исполнители, сроки), варианты процессов |
| Проводник | Обложки (стартовые страницы), видимость папок Входящие/Исходящие |
| Списки | Колонки, сортировка, панели действий, фильтры |
| Карточки | Расположение полей, добавление/скрытие элементов формы |
| Диалоги | Пошаговые диалоги сбора данных с логикой |

## Бизнес-процессы: базовые блоки

Схемы строятся из библиотеки базовых блоков:

| Блок | Описание |
|------|----------|
| Конец ветки | Завершает текущую ветку, не останавливая остальные |
| И / ИЛИ | Объединяют несколько переходов логическими условиями |
| Условие | Добавляет условное ветвление |
| Задание | Создаёт задание для исполнителя |
| Уведомление | Создаёт уведомление |
| Ожидание | Приостанавливает задачу на заданный период |
| Ожидание с условием | Мониторит условие и продолжает при его выполнении |
| Настройка прав доступа | Выдаёт или снимает права на объекты |
| Задача | Создаёт подзадачу |
| Скрипт | Выполняет код разработчика |
| Мониторинг | Отслеживает наступление события |

**Важно:** блоки Задача, Скрипт и Мониторинг не входят в базовую библиотеку — разработчики создают прикладные блоки на их основе в DDS.

## Варианты процессов

Сложный процесс можно разделить на **варианты**, когда:
- Состав/порядок блоков существенно отличается по входным данным
- Нужно менять один процесс, не затрагивая другие

Каждый вариант имеет:
- **Критерии выбора** — условия автоматического выбора варианта
- **Приоритет** — порядок при совпадении нескольких критериев

```
Пример: Согласование договоров
├── Вариант 1: Сумма > 1M → согласование CEO + СБ
├── Вариант 2: Сумма > 100K → согласование CEO
└── Вариант 3: По умолчанию → стандартное согласование
```

## Вычисляемые роли

Используются когда исполнитель определяется динамически:
- Зависит от свойства сущности, связанной с задачей
- Выбор исполнителя зависит от набора условий

### Пример

```
Ситуация: 4 организации, у каждой свой бухгалтер.
Решение:
1. Создать вычисляемую роль "Бухгалтер по организации" (контекст = "Наша организация")
2. В правилах вычисления — назначить бухгалтеров по организациям
3. В блоке «Задание» указать эту роль как исполнителя
```

**Преимущества:**
- Убирает лишние блоки «Условие» из схемы
- Состав можно расширять без изменения схемы
- Изменения роли применяются к уже запущенным задачам (для ещё не достигнутых блоков)

## Настройка списков

### Колонки (табличный вид)

- Включение/выключение видимости колонок (чекбоксы)
- Перетаскивание для изменения порядка
- Изменение ширины перетягиванием границ
- Сортировка по умолчанию (клик → прямая, повторный клик → обратная)

### Компактный вид

- До 4 элементов на запись
- Фиксированные элементы для задач: Тема, Срок, От, Создано, Изменено
- Фиксированные элементы для документов: Имя, Создано, Изменено

### Группировка и двухстрочный режим

- **Двухстрочный режим** — записи в две строки
- **Область группировки** — группировка по значениям колонок

### Панель действий списка

| Тип элемента | Описание |
|-------------|----------|
| Группа | Логический контейнер с разделителями |
| Кнопка | Вызов одного действия |
| Кнопка-меню | Выпадающее меню из нескольких действий |

**Свойства видимости элементов:**
- `Отображать на панели действий`
- `Отображать только иконку` — экономия места
- `Редко используемое действие` — скрывается в «ещё»
- `Отображать в контекстном меню`

### Панель фильтров

Показать/скрыть отдельные параметры фильтрации или целые группы. Для Входящих/Исходящих — дополнительно экспресс-фильтры: «Новые», «В работе», «30 дней».

## Настройка карточек

### Элементы панели действий

**Кнопка:**
- Действие, отображаемое имя, иконка
- Флаги: основное действие, редко используемое

**Кнопка-меню:** группирует несколько действий в выпадающий список.

**Кнопка-переключатель:** скрывает/показывает элементы формы. Работает с действиями с флагом «Toggle action». Сбрасывается при закрытии карточки.

**Кнопка-переключатель состояния:** «залипающая» кнопка, меняющая boolean-свойство. Состояние сохраняется между открытиями карточки. Свойство: `Свойство состояния` (только boolean).

### Предпросмотр и вложения

Для задач/заданий/уведомлений:
- Позиция панели предпросмотра (справа / по центру)
- Видимость: обе панели / только вложения / только предпросмотр

### Табличные части

Добавляются как «Простой контрол» → привязка к коллекционному свойству → тип «Редактор коллекционного свойства».

## NoCode-диалоги

Интерактивные окна для пошагового сбора данных от пользователей.

### Возможности

- Поля показываются/скрываются по значениям других полей
- Единая точка входа для различных запросов сотрудников
- По введённым данным выполняется одно из настроенных действий

### Настройка

1. Создать диалог в справочнике «Диалоги»
2. Настроить карточку и действия (шаги, переходы, завершение)
3. Добавить диалог на обложку модуля
4. Протестировать кнопкой «Запустить»

Права доступа: пользователи видят диалоги через операцию «Запуск диалога».

## Изменение свойств при старте блока

Автоматическое изменение свойств объектов при старте блока процесса.

| Поле | Описание |
|------|----------|
| `*Имя` | Название операции |
| `*Что изменить` | Вычисляемое выражение → свойство объекта |
| `*Операция` | «Установить значение» или «Очистить» |
| `*Значение` | Выбрать из справочника, вычисляемое выражение или ввести вручную |

---

## Просмотр состояния блоков (отладка)

Только для членов роли «Администраторы»:

1. Отправить тестовую задачу по проверяемому варианту процесса
2. В карточке задачи → «Показать схему»
3. Проверить значения свойств блоков и параметров процесса
4. Принудительно выполнить блоки:
   - «Задание» — завершить по сроку
   - «Ожидание» — пропустить период
   - «Ожидание с условием» — проверить условие вручную

---

## Экспорт/импорт настроек

Все NoCode-настройки (списки, формы, процессы) можно экспортировать и импортировать между экземплярами системы (тест → продуктив).

Типы настроек:
- **По умолчанию** — для всех пользователей
- **Пользовательские** — индивидуальные настройки конкретного пользователя
