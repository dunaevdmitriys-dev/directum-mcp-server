## Что такое решение на Sungero

Sungero — платформа для создания корпоративных систем управления документами и бизнес-процессами. Пользовательская разработка — это набор **модулей**, которые подключаются к базовому решению DirectumRX.

```
DirectumRX (базовое решение)
  └── Ваши модули (прикладная разработка)
        ├── MyModule1 (сущности, функции, маршруты)
        ├── MyModule2
        └── ...
```

---

## Архитектура сервисов

Платформа состоит из независимых сервисов, взаимодействующих через очередь сообщений (RabbitMQ):

| Сервис | Назначение |
|--------|-----------|
| `SungeroWebServer` | Веб-сервер, обрабатывает запросы браузера |
| `SungeroWebClient` | Фронтенд (SPA на JavaScript) |
| `SungeroWorker` | Выполнение фоновых задач (.NET) |
| `SungeroWorkerNetFx` | Фоновые задачи на .NET Framework |
| `JobScheduler` | Планировщик фоновых заданий |
| `WorkflowProcessService` | Обработка бизнес-процессов (workflow) |
| `WorkflowBlockService` | Выполнение блоков схем |
| `IndexingService` | Полнотекстовая индексация (Elasticsearch) |
| `StorageService` | Хранение файлов документов |
| `PreviewService` | Генерация превью документов |
| `ReportService` | Генерация отчётов |
| `IntegrationService` | Интеграция с внешними системами |
| `SungeroDB` | Управление базой данных (PostgreSQL) |
| `KeyDerivationService` | Управление ключами шифрования |
| `LogService` | Централизованное логирование |

## Трёхуровневая модель кода

Каждый модуль содержит три типа кода:

### Server (серверный)
- Выполняется на **веб-сервере**
- Доступно: работа с БД, создание/удаление сущностей, файловая система
- Недоступно: диалоги, UI-взаимодействие
- Примеры: бизнес-логика, инициализация данных, получение списков

### Client (клиентский)
- Выполняется в контексте **запроса пользователя** (веб-клиент → сервер)
- Доступно: диалоги, окна сообщений, управление формой (IsVisible, IsEnabled)
- Недоступно: прямые SQL-запросы, прямое создание/удаление сущностей
- Примеры: события форм, действия пользователя, валидация UI

### Shared (разделяемый)
- Выполняется **в обоих контекстах** — доступен и серверу и клиенту
- Ограничения: нет диалогов, нет прямой работы с сущностями
- Примеры: вычисление дат, форматирование строк, константы, валидация значений

```
Правило: "Shared ⊆ Client, Shared ⊆ Server, но Client ≠ Server"

Если нужно вызвать серверную функцию из клиента → атрибут [Remote]
```

## Структура модуля

```
MyModule/
  ├── Entities/                     # Сущности (типы данных)
  │   ├── MyEntity/
  │   │   ├── MyEntity.cs           # Основной файл сущности
  │   │   ├── Server/               # Серверные события и функции
  │   │   │   ├── MyEntityServerHandlers.cs
  │   │   │   └── MyEntityServerFunctions.cs
  │   │   ├── Client/               # Клиентские события
  │   │   │   └── MyEntityClientHandlers.cs
  │   │   └── Shared/               # Разделяемые обработчики
  │   │       └── MyEntitySharedHandlers.cs
  │   └── ...
  ├── ModuleFunctions.cs            # Функции модуля
  ├── Server/
  │   └── ModuleServerFunctions.cs  # Серверные функции модуля
  ├── Client/
  │   └── ModuleClientFunctions.cs  # Клиентские функции
  ├── Shared/
  │   └── ModuleSharedFunctions.cs  # Разделяемые функции
  ├── Reports/                      # Отчёты
  ├── Resources/                    # Ресурсы (строки, изображения)
  ├── Constants/                    # Константы
  ├── Structures/                   # Структуры данных
  └── Initialization/               # Инициализация модуля
      └── ModuleInitializer.cs
```

## Жизненный цикл разработки

```
1. DDS (Directum Development Studio)
   ↓ создание/редактирование модуля
2. Publish (публикация)
   ↓ компиляция C# → .dll сборки
3. Sungero Web Server
   ↓ загрузка сборок
4. Тестирование в браузере
```

**Development Studio (DDS)** — IDE для разработки на Sungero. Визуальные редакторы для:
- Создания сущностей и их свойств
- Настройки форм отображения
- Редактирования C# кода (сервер/клиент/shared)
- Создания маршрутов бизнес-процессов

## Зависимости модулей

Модули могут зависеть друг от друга. Зависимость объявляется в манифесте модуля:

```json
{
  "dependencies": [
    "Sungero.Docflow",
    "Sungero.Company",
    "Sungero.Contracts"
  ]
}
```

Стандартные модули платформы:

| Модуль | Namespace | Назначение |
|--------|-----------|-----------|
| Docflow | `Sungero.Docflow` | Документооборот |
| Company | `Sungero.Company` | Сотрудники, подразделения |
| Parties | `Sungero.Parties` | Контрагенты |
| Contracts | `Sungero.Contracts` | Договоры |
| Workflow | `Sungero.Workflow` | Задачи, задания |
| CoreEntities | `Sungero.CoreEntities` | Пользователи, роли, группы |
| Exchange | `Sungero.Exchange` | ЭДО обмен |
| SmartProcessing | `Sungero.SmartProcessing` | Умная обработка (Ario) |

## Мультитенантность

Платформа поддерживает несколько тенантов (компаний) в одной инсталляции. Каждый тенант имеет:
- Свою БД (или схему в общей БД)
- Своих пользователей
- Свои настройки
- Общие прикладные модули (код)

Тенанты управляются через `rxcmd` (командная строка) или `platform_plugin` (Python).

## Ключевые концепции

| Концепция | Описание |
|-----------|----------|
| Entity | Объект данных с полями, методами, событиями |
| Repository | Статический класс для CRUD-операций (`Employees.Create()`, `Employees.GetAll()`) |
| PublicFunctions | Публичные функции модуля, доступные извне |
| Remote | Атрибут для вызова серверного метода из клиентского кода |
| Scheme | Схема бизнес-процесса (граф блоков) |
| ProcessKind | Вариант процесса — конфигурация маршрута для конкретного вида документа |
| Override | Перекрытие метода базового решения в прикладной разработке |
