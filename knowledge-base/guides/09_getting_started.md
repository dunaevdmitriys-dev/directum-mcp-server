# Создание решения с нуля (Getting Started)

> Источник: `sds_protcess_razrabotki.htm`, `sds_elementy_razrabotki.htm`, `sds_sozdanie_svoistva.htm`, `sds_sozdanie_deistviia.htm`, `sds_publikatsiya.htm`

---

## Обзор процесса разработки

```
1. Запустить среду разработки (DDS)
   ↓ получить последнюю версию кода с Git
2. Создать решение → модули → типы сущностей
   ↓ написать код (Server / Client / Shared)
3. Собрать решение (компиляция)
   ↓ зафиксировать и отправить в Git
4. Опубликовать в локальную систему
   ↓ тестирование в браузере
5. Создать пакет разработки
   ↓ публикация в продуктивную систему через Directum Launcher
```

---

## Среда разработки (DDS)

**Directum Development Studio** — IDE для разработки решений на платформе Sungero. Предоставляет:
- Визуальный редактор сущностей и их свойств
- Редактор форм (карточек) сущностей
- Редактор кода C# (серверный / клиентский / разделяемый)
- Редактор схем бизнес-процессов
- Инструменты сборки, публикации и отладки

### Ключевые кнопки панели инструментов

| Кнопка | Действие |
|--------|----------|
| Получить изменения с сервера | Обновить исходный код из Git |
| Собрать | Скомпилировать все решения |
| Опубликовать... | Публикация в локальную систему |
| Опубликовать выборочно... | Публикация выбранных решений |

## Шаг 1: Создание решения

Решение — контейнер верхнего уровня для ваших модулей. В решении вы:
- Создаёте собственные модули
- Перекрываете модули и типы сущностей базового решения (DirectumRX)

```
MySolution (решение)
  ├── MyModule1 (модуль)
  │   ├── Entities/       (типы сущностей)
  │   ├── Server/          (серверные функции)
  │   ├── Client/          (клиентские функции)
  │   ├── Shared/          (разделяемые функции)
  │   ├── Reports/         (отчёты)
  │   ├── Resources/       (локализованные строки)
  │   └── Initialization/  (инициализация данных)
  └── MyModule2 (модуль)
      └── ...
```

В DDS: **Файл → Создать решение**. Укажите имя (латиницей, PascalCase).

---

## Шаг 2: Создание модуля

Модуль — основная единица разработки. Содержит сущности, функции, отчёты, ресурсы.

В DDS: щелчок правой кнопкой на решении → **Добавить модуль**. Укажите:
- **Имя** — латиницей, PascalCase (например, `PurchaseManagement`)
- **Зависимости** — от каких модулей зависит ваш модуль

### Стандартные модули платформы (зависимости)

| Модуль | Namespace | Что даёт |
|--------|-----------|----------|
| Docflow | `Sungero.Docflow` | Документооборот, согласование |
| Company | `Sungero.Company` | Сотрудники, подразделения, НашиОрганизации |
| Parties | `Sungero.Parties` | Контрагенты, персоны |
| Contracts | `Sungero.Contracts` | Договоры |
| Workflow | `Sungero.Workflow` | Задачи, задания |
| CoreEntities | `Sungero.CoreEntities` | Пользователи, роли, группы |

---

## Шаг 3: Создание типа сущности

Тип сущности — объект данных (справочник, документ, задача, задание).

### Выбор базового типа

| Вам нужен | Базовый тип | Namespace |
|-----------|-------------|-----------|
| Справочник (простой) | `DatabookEntry` | `Sungero.CoreEntities` |
| Документ | `OfficialDocument` | `Sungero.Docflow` |
| Договор | `ContractualDocument` | `Sungero.Contracts` |
| Задача | `Task` | `Sungero.Workflow` |
| Задание | `Assignment` | `Sungero.Workflow` |
| Уведомление | `Notice` | `Sungero.Workflow` |

В DDS: щелчок правой кнопкой на модуле → **Добавить тип сущности** → выбрать базовый тип.

### Наследование vs Перекрытие

| Наследование | Перекрытие |
|-------------|-----------|
| Создаёт **новый** тип на основе существующего | **Изменяет** поведение существующего типа |
| Новый тип в дереве сущностей | Тот же тип, но с дополнительной логикой |
| `class MyContract : Contract` | `override void BeforeSave()` в существующем контракте |
| Для новых видов документов | Для изменения логики базового решения |

**Правило:** если нужен новый вид — наследование. Если нужно изменить существующий — перекрытие.

## Шаг 4: Добавление свойств

В редакторе типа сущности → узел **«Свойства»** → кнопка **Добавить → Свойство**.

### Типы свойств

| Тип | C# тип | Описание |
|-----|--------|----------|
| Строка | `string` | Текст (длина настраивается) |
| Целое | `int?` | Целое число |
| Вещественное | `double?` | Число с плавающей точкой |
| Дата | `DateTime?` | Дата и время |
| Логическое | `bool?` | Да / Нет |
| Перечисление | `Enumeration?` | Набор фиксированных значений |
| Ссылка | `IEntity` | Ссылка на другую сущность |
| Коллекция | `ICollection` | Табличная часть (список строк) |

### Добавление простого свойства

1. В узле «Свойства» → **Добавить → Свойство**
2. Задайте настройки:
   - **Имя** (латиницей, PascalCase)
   - **Тип данных**
   - **Обязательность** (IsRequired)
   - **Значение по умолчанию**

### Добавление свойства-коллекции

1. **Добавить → Коллекцию** → укажите имя
2. Выделите коллекцию → **Добавить → Свойство в коллекцию**
3. Задайте свойства строк коллекции

### Добавление свойства на форму

После создания свойства добавьте его на карточку сущности:
- В редакторе типа сущности → узел **«Форма»**
- Перетащите свойство на нужную позицию

## Шаг 5: Создание действий (кнопок)

Действие — кнопка на ленте карточки или списка сущности.

В редакторе типа сущности → узел **«Действия»** → кнопка **Добавить**.

Параметры действия:
- **Имя** — латиницей (например, `CreateReport`)
- **Заголовок** — текст кнопки (локализуется)
- **Иконка** — значок на ленте

### Добавление действия на ленту

В узле **«Лента»** перетащите действие в нужную группу ленты.

### Обработчики событий действия

```csharp
// Проверка возможности выполнения (серый/активный).
public virtual bool CanCreateReport(Sungero.Domain.Client.CanExecuteActionArgs e)
{
  return _obj.Status == MyEntity.Status.Active;
}

// Выполнение действия.
public virtual void CreateReport(Sungero.Domain.Client.ExecuteActionArgs e)
{
  // Вызов серверной функции через Remote.
  var result = Functions.Module.Remote.GenerateReport(_obj.Id);
  if (result != null)
    result.ShowCard();
}
```

## Шаг 6: Написание кода

### Серверные функции модуля

В редакторе модуля → группа «Функции» → **Добавить** рядом с серверной функцией.

```csharp
/// <summary>
/// Создать организацию.
/// </summary>
[Remote]
public static ICompany CreateCompany()
{
  return Companies.Create();
}
```

### Клиентские функции модуля

```csharp
/// <summary>
/// Клиентская функция: создать и показать организацию.
/// </summary>
public static void CreateCompany()
{
  Functions.Module.Remote.CreateCompany().Show();
}
```

### Формат вызова функций

| Контекст вызова | Формат |
|----------------|--------|
| Из того же модуля (сервер) | `Functions.Module.MyFunction(params)` |
| Из клиента → сервер (Remote) | `Functions.Module.Remote.MyFunction(params)` |
| Из другого модуля ([Public]) | `OtherModule.PublicFunctions.Module.MyFunction(params)` |
| Из другого модуля (Remote + Public) | `OtherModule.PublicFunctions.Module.Remote.MyFunction(params)` |
| Из функции того же типа сущности | `MyFunction(params)` (без `Functions.Module`) |

### Функции типа сущности

В редакторе типа сущности → группа «Функции» → **Добавить**.

```csharp
// Серверная функция типа сущности.
[Public]
public virtual decimal CalculateTotal()
{
  return _obj.Lines.Sum(l => l.Amount ?? 0);
}
```

Формат вызова функций типа сущности:

| Контекст | Формат |
|----------|--------|
| Изнутри того же типа | `this.CalculateTotal()` или просто `CalculateTotal()` |
| Из другого места (сервер) | `Functions.MyEntity.CalculateTotal(entity)` |
| Из клиента (Remote) | `Functions.MyEntity.Remote.CalculateTotal(entity)` |
| Из другого модуля | `MyModule.PublicFunctions.MyEntity.CalculateTotal(entity)` |

### Атрибуты функций

| Атрибут | Описание |
|---------|----------|
| `[Public]` | Функция доступна из других модулей |
| `[Remote]` | Серверная функция, вызываемая из клиентского кода |
| `[Remote(IsPure = true)]` | Remote без побочных эффектов (только чтение) |

## Шаг 7: Сборка и публикация

### Сборка

Кнопка **Собрать** на панели инструментов. Проверяет компиляцию C# кода.

### Публикация в локальную систему

1. Кнопка **Опубликовать...** → выбрать решения для публикации
2. Создаётся пакет развёртывания (не более 1 ГБ)
3. Пакет отправляется на веб-сервер
4. Обновляются структуры БД
5. Запускается механизм обновления прикладной разработки
6. Веб-сервер и сервисы перезапускаются (если менялся не только код и ресурсы)
7. Открывается веб-клиент для тестирования

**Важно:** перед публикацией создайте резервную копию БД.

### Публикация в продуктивную систему

1. В DDS: **Сборка → Экспортировать пакет разработки**
2. В **Directum Launcher**: страница «Публикация» → указать пакет → опубликовать

**Важно:** публикуйте в продуктивную систему в нерабочее время. Веб-сервер перезагружается и недоступен для пользователей.

## Шаг 8: Отладка

### Отладка серверного кода

1. В DDS: **Сборка → Присоединить к процессу**
2. Выберите процесс `SungeroWebServer` (или `SungeroWorker` для фоновых заданий)
3. Установите точки останова в серверном коде
4. Выполните действие в веб-клиенте — отладчик остановится на точке

### Отладка клиентского кода

1. Присоединитесь к процессу `SungeroWebServer`
2. Установите точки останова в клиентском коде
3. Откройте карточку сущности в браузере — при событии (Showing, Refresh) отладчик сработает

### Инструменты отладки

| Инструмент | Описание |
|-----------|----------|
| Точки останова (Breakpoints) | Остановка выполнения на строке кода |
| Наблюдение (Watch) | Просмотр значений переменных |
| Пошаговое выполнение (Step) | F10 — следующая строка, F11 — вход в метод |
| Стек вызовов (Call Stack) | Цепочка вызовов до текущей точки |

## Шаг 9: Инициализация данных модуля

Метод `Initializing` вызывается при первом запуске или после обновления модуля.

```csharp
public override void Initializing(Sungero.Domain.ModuleInitializingEventArgs e)
{
  CreateDefaultDocumentKinds();
  CreateDefaultRoles();
  GrantDefaultRights();
}

public static void CreateDefaultDocumentKinds()
{
  InitializationLogger.Debug("Init: Creating default document kinds.");
  CreateDocumentKindIfNotExists("Закупочный договор");
  CreateDocumentKindIfNotExists("Входящее письмо");
}

// Паттерн CreateIfNotExists — не дублировать при повторной инициализации.
public static void CreateDocumentKindIfNotExists(string name)
{
  var existing = Docflow.DocumentKinds.GetAll()
    .FirstOrDefault(k => k.ShortName == name);

  if (existing != null)
    return;

  var kind = Docflow.DocumentKinds.Create();
  kind.ShortName = name;
  kind.Name = name;
  kind.Save();
}
```
