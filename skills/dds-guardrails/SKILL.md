# DDS Guardrails — Разрыв цикла ошибок

> Все правила DDS — в CLAUDE.md (18 пунктов). Этот файл содержит ТОЛЬКО то, чего нет в CLAUDE.md: поведенческие антипаттерны Claude и алгоритм их предотвращения.

## Алгоритм разрыва цикла

```
Claude предложил fix → fix не помог → Claude предлагает ТОТ ЖЕ fix → ЦИКЛ

Разрыв:
1. СТОП. Не повторять тот же fix.
2. Прочитай сообщение об ошибке ЦЕЛИКОМ (не первую строку).
3. Запусти /validate-all — найди ВСЕ проблемы разом.
4. Сверь с CLAUDE.md «Known Issues» (пункты 1-18).
5. Исправь ВСЕ найденные проблемы ОДНИМ набором изменений.
6. Проверь через /validate-all повторно.
7. Если снова не помогло → MCP: trace_errors → спроси пользователя.
```

## Поведенческие антипаттерны Claude

Это НЕ правила DDS (они в CLAUDE.md), а типичные ПОВЕДЕНЧЕСКИЕ ошибки Claude при работе с Directum RX.

### АНТИПАТТЕРНЫ ИЗ PRODUCTION (Targets/OmniApplied)

### 11. «Override модуля без base.Method()»
**Паттерн:** Перекрыть метод платформенного модуля без вызова `base.Method()`.
**Почему нельзо:** Ломает цепочку перекрытий — платформенная логика не выполняется, данные теряются.
**Вместо этого:** Всегда `base.Method()` первой строкой.
**Reference:** `omniapplied/source/Sungero.Omni/Sungero.Omni.Server/Sungero.Company/ModuleServerFunctions.cs`

### 12. «AsyncHandler без retry-стратегии»
**Паттерн:** `catch (Exception) { Logger.Error(...); }` — залогировал и забыл, без `args.Retry`.
**Почему нельзя:** Данные остаются несогласованными навсегда — АО больше не запустится.
**Вместо этого:** 4 production-паттерна retry:
- Отложенный: `args.NextRetryTime = Calendar.Now.AddHours(12)` (из KPI DeleteCheckFile)
- Batch: `Locks.TryLock` + обработка по 100 + retry пока есть необработанные (из KPI ImportActualValues)
- Fan-out: родитель создаёт дочерние АО через `ExecuteAsync` (из Targets ConvertTargetsMapsDates)
- Bounded: `args.Retry = args.RetryIteration < args.MaxRetryCount` (из Targets GrantAccessForProjectPlans)
**Reference:** `targets/CODE_PATTERNS_CATALOG.md` секция 3

### 13. «Initializer без версионирования»
**Паттерн:** Вся логика в `Initializing()` одним блоком — без `ModuleVersionInit`.
**Почему нельзя:** При обновлении пакета инициализация запустится повторно — дубликаты ролей, справочников, прав.
**Вместо этого:** Версионный паттерн `ModuleVersionInit` (см. DTCommons — 5 версий: 1.1→1.5).
**Reference:** `targets/source/DirRX.DTCommons/DirRX.DTCommons.Server/ModuleInitializer.cs`

### ПОВЕДЕНЧЕСКИЕ АНТИПАТТЕРНЫ CLAUDE

### 1. «Быстрее руками»
**Паттерн:** Редактировать .csproj, .g.cs или другие DDS-managed файлы напрямую.
**Почему нельзя:** DDS перезатрёт при переоткрытии. Часы работы пропадут.
**Вместо этого:** Сторонние библиотеки → DDS UI. Структуры → Module.mtd. Генерируемый код → `chmod -w`.

### 2. «Добавлю локализацию потом»
**Паттерн:** Создать .mtd без System.resx, отложить на потом.
**Почему нельзя:** Пользователь увидит пустые подписи СЕЙЧАС. Забудешь — баг уйдёт в прод.
**Вместо этого:** Создавать System.ru.resx ОДНОВРЕМЕННО с .mtd. Это часть одного артефакта.

### 3. «Скопирую формат из существующего файла»
**Паттерн:** Копировать resx-ключи из стороннего пакета (CRM v2.0 и др.).
**Почему нельзя:** Чужой пакет может содержать `Resource_<GUID>` вместо `Property_<Name>`.
**Вместо этого:** Всегда генерировать ключи по конвенции CLAUDE.md #10. Не доверять чужим примерам.

### 4. «Это логически связанные данные — нужна коллекция»
**Паттерн:** Добавлять CollectionPropertyMetadata в DatabookEntry.
**Почему нельзя:** DDS 26.1 не поддерживает. "Missing area" + NullReferenceException.
**Вместо этого:** Дерево решений: DatabookEntry = простые свойства, Document = коллекции.

### 5. «Перекрою Controls чтобы настроить позже»
**Паттерн:** `"Overridden": ["Controls"]` с `"Controls": []`.
**Почему нельзя:** Пустой массив = пустая карточка. Без полей.
**Вместо этого:** Либо не перекрывать, либо заполнить Controls полностью.

### 6. «New — логичное имя для статуса»
**Паттерн:** Именовать enum values/свойства зарезервированными словами C#.
**Почему нельзя:** Ошибка компиляции сгенерированного кода.
**Вместо этого:** `NewDeal`, `DefaultPriority`, `EventType`. Проверять по списку CLAUDE.md #4.

### 7. «Deal — чистое имя»
**Паттерн:** Одинаковый Code у свойств в иерархии наследования.
**Почему нельзя:** Конфликт колонок в БД.
**Вместо этого:** Префикс модуля: `CrmDeal`, `InvDeal`.

### 8. «Всё должно быть в DDS для единообразия»
**Паттерн:** Создавать лёгкие CRM-сущности (Message, CallLog) как DDS DatabookEntry.
**Почему нельзя:** Оверхед DDS, проблемы импорта, ненужная сложность.
**Вместо этого:** Лёгкие данные → CrmApiV3 + PostgreSQL. DDS → только для сущностей с формами/workflow.

### 9. «DDS подхватит изменения автоматически»
**Паттерн:** Изменить .mtd и не перезапустить DDS.
**Почему нельзя:** DDS кэширует .mtd при старте.
**Вместо этого:** Перезапуск DDS после каждого изменения .mtd.

### 10. «DocumentType более описательно чем Type»
**Паттерн:** Использовать в JSON-сериализаторе имена, отличающиеся от Module.mtd.
**Почему нельзя:** Десериализация упадёт, поля будут null.
**Вместо этого:** Имена в сериализаторе === имена в Module.mtd PublicStructures. Точно.
