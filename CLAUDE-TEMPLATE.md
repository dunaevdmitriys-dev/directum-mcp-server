# Directum RX — Рабочее пространство

## Кто и что
Платформа Directum RX 26.1.
Язык общения — русский.
Полная автономность — не задавай лишних вопросов, делай.

## АЛГОРИТМ: Сначала найди, потом пиши

> **ОБЯЗАТЕЛЬНО.** Перед созданием любого файла (.mtd, .resx, .cs):

1. **Найти аналог** — `search_metadata name=<похожая_сущность>` или Grep по source/
2. **Прочитать найденный .mtd ЦЕЛИКОМ** — понять структуру, GUID, связи
3. **Адаптировать** — копировать структуру, менять имена/GUID. НЕ генерировать из головы
4. **Валидация** — после создания -> `/validate-all`

## ЗАПРЕЩЕНО
- Генерировать GUID из головы — только из существующих .mtd
- Придумывать имена OData EntitySet — использовать `predict_odata_name`
- Создавать .mtd без чтения аналога — ВСЕГДА сначала найти пример
- Ставить `Resource_<GUID>` в .resx — только `Property_<Name>`
- Называть enum зарезервированными словами: New, Class, Default, Event
- Делать вложенные таблицы (CollectionProperty) в справочниках (DatabookEntry)
- Использовать `DateTime.Now` — только `Calendar.Now`
- Использовать обычный `class` — только `partial class`
- Пропускать валидацию — после КАЖДОГО изменения -> `/validate-all`
